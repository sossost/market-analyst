## 검증 항목

### 1. 데이터 정합성 (0~10점)
- EPS/매출 수치, 등급 판정 기준이 서술과 일치하는가?
- SEPA 기준 적용 오류 여부 (EPS/매출 YoY >25%, 가속, 마진확대)
- 분기별 실적 테이블과 펀더멘탈 분석 서술 간 수치 불일치 감점
- 판단 근거를 1~2줄로 명시

### 2. 서사 근거 (0~10점)
- LLM 분석(narrative) 섹션이 정량 데이터에 기반하는가?
- 데이터 없이 낙관 서사를 전개하는 경우 감점
- 수치를 인용하며 근거를 제시하는가?
- 판단 근거를 1~2줄로 명시

### 3. 구조 완결성 (0~10점)
- 필수 섹션 포함 여부: 기술적 현황 / 펀더멘탈 분석 / 분기별 실적 / 애널리스트 분석 / 종합 판단
- 각 섹션이 비어 있거나 형식적으로만 채워진 경우 감점
- 판단 근거를 1~2줄로 명시

### 4. 투자 판단 명확성 (0~10점)
- S등급 종목으로서 왜 주목해야 하는지 명확히 서술되었는가?
- "상승 가능성 있음"처럼 모호한 결론 감점
- 구체적 판단 기준(예: "EPS 3분기 연속 가속", "섹터 Phase 2 진입")이 제시되었는가?
- 판단 근거를 1~2줄로 명시

## 출력 형식

반드시 아래 JSON만 출력하세요. 코드 펜스 없이 순수 JSON.

{
  "scores": {
    "dataAccuracy": <0~10>,
    "narrativeBasis": <0~10>,
    "structuralCompleteness": <0~10>,
    "investmentClarity": <0~10>
  },
  "totalScore": <0~40>,
  "hasIssue": <true|false>,
  "issueTitle": "<GitHub 이슈 제목. hasIssue=false면 빈 문자열>",
  "issueBody": "<GitHub 이슈 본문 (마크다운). hasIssue=false면 빈 문자열>",
  "summary": "<검증 요약 1~2줄>"
}

판단 기준:
- hasIssue = true 조건: 어느 하나라도 7점 미만이거나 totalScore ≤ 28
- issueBody에는 감점 항목별 근거, 재발 방지 제안 포함
- 모든 점수 ≥ 7이고 totalScore ≥ 30이면 hasIssue = false
