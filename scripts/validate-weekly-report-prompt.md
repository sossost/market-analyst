## 검증 항목

### 1. 팩트 일관성 (0~10점)
- 데이터 수치와 서술이 일치하는가?
- 예: "섹터 RS 상승" 서술인데 실제 RS 수치가 하락인 경우 → 감점
- 판단 근거를 1~2줄로 명시

### 2. bull-bias 필터 (0~10점)
- 리스크/약세 언급 비율이 충분한가? (목표: 낙관/비관 언급 비율 ≤ 70:30)
- 데이터 없이 낙관 결론을 내리는가?
- 판단 근거를 1~2줄로 명시

### 3. 구조/가독성 (0~10점)
- 리포트 포맷 준수 여부 (시장 흐름 → 섹터 분석 → 종목 추천 순서)
- 핵심 정보가 상단에 있는가?
- 판단 근거를 1~2줄로 명시

### 4. 이전 대비 변화 (0~10점)
- 직전 주간 리포트 대비 복붙 수준으로 동일한 문장이 반복되는가?
- 새로운 인사이트가 포함되었는가?
- 직전 리포트 없으면 이 항목은 null로 표시
- 판단 근거를 1~2줄로 명시

### 5. 전망의 검증 가능성 (0~10점)
- 예측/전망이 구체적이고 검증 가능한 조건으로 표현되었는가?
- "상승 가능성 있음" 같은 모호한 서술은 감점
- "OO 섹터가 2주 내 RS 70 이상 유지 시 Phase 2 진입 예상" 같은 구체적 조건은 가점
- 판단 근거를 1~2줄로 명시

## 출력 형식

반드시 아래 JSON만 출력하세요. 코드 펜스 없이 순수 JSON.

{
  "scores": {
    "factConsistency": <0~10>,
    "bullBias": <0~10>,
    "structure": <0~10>,
    "novelty": <0~10 또는 null>,
    "verifiability": <0~10>
  },
  "totalScore": <0~50>,
  "hasIssue": <true|false>,
  "issueTitle": "[주간 QA] {REPORT_DATE} — {핵심 1줄 요약}  (hasIssue=false면 빈 문자열)",
  "issueBody": "<아래 고정 템플릿을 반드시 따를 것. hasIssue=false면 빈 문자열>",
  "summary": "<검증 요약 1~2줄>"
}

판단 기준:
- hasIssue = true 조건: 어느 하나라도 7점 미만이거나 totalScore ≤ 35
- novelty가 null이면 totalScore는 나머지 4항목 합산으로만 판단 (28점 이하면 hasIssue)
- 모든 점수 ≥ 7이고 totalScore ≥ 36이면 hasIssue = false

issueTitle 형식 (반드시 준수):
- 형식: [주간 QA] YYYY-MM-DD — {핵심 1줄 요약}
- 예시: [주간 QA] 2026-03-22 — 전망 검증 가능성 부족, 팩트 불일치
- {REPORT_DATE} 자리에 실제 리포트 날짜를 넣을 것
