# Market Analyst — Roadmap

**최종 업데이트:** 2026-03-08

---

## 최종 목적지

**시간이 지날수록 똑똑해지는 시장 분석 시스템.**

구체적으로:
1. 매일 미국 시장 전체를 스캔하여 **상승 초입(Phase 2) 주도섹터/주도주를 남들보다 먼저 포착**
2. 예측을 기록하고 검증하여 **어떤 조건에서 맞았고 틀렸는지 패턴을 축적**
3. 축적된 패턴이 다음 분석에 반영되어 **시스템의 판단 정확도가 시간과 함께 향상**

이것은 "LLM이 시장을 이해하는 시스템"이 아니다.
**데이터 파이프라인 + 패턴 축적 + 규율 있는 추적**이 만드는 복리 효과를 추구하는 시스템이다.

### 이 시스템이 아닌 것
- 트레이딩 시그널 생성기 (목표가/손절가 없음)
- 단기 매매 도구 (구조적 변화의 초기 신호가 목표)
- LLM의 "직관"에 의존하는 시스템 (데이터가 말하게 한다)

### 성공의 정의

| 기준 | 목표 | 측정 방법 |
|------|------|----------|
| Phase 2 사전 포착 | 컨센서스보다 1-2주 먼저 | 추천일 vs 실제 상승 시작일 비교 (추천 10건+ 후) |
| 학습 패턴 축적 | 적중률 60%+ 학습 10개 이상 | `agent_learnings` 테이블 (6개월 후) |
| 리포트 활용도 | 실제 투자 판단에 월 1건+ 활용 | **월 1회 수신자 피드백** — 리포트별 유용성 3점 척도(활용함/참고함/무시함) + 자유 코멘트. Discord DM 또는 간단 폼으로 수집 |

## 현재까지 완료한 것

### Layer 1: 데이터 인프라 (F1) — Done
시스템의 기반. 매일 미국 전체 종목을 스캔하고 구조화된 데이터를 생산.

| 구성 요소 | 설명 |
|-----------|------|
| Stock Phases | Weinstein Stage Analysis — 8개 조건으로 Phase 1~4 판별 |
| Sector/Industry RS | 상대 강도 점수 + 가속도 + 브레드스 (Phase 2 비율) |
| 펀더멘탈 데이터 | 분기 재무제표 + 비율 (FMP API) |
| 돌파/노이즈 신호 | 기술적 시그널 필터링 |

**파이프라인:** GitHub Actions, 일~금 UTC 23:30 (KST 08:30), ~50분 소요.

### Layer 2: 에이전트 코어 (F2 + F5) — Done
데이터를 읽고 리포트를 생성하는 Claude 에이전트.

| 구성 요소 | 설명 |
|-----------|------|
| Agent Loop | Claude API + 13개 도구, 일간/주간 분리 |
| 리뷰 에이전트 | 리포트 품질 검증 파이프라인 |
| 딜리버리 | Discord + Gist, 일간/주간 스케줄 |

### Layer 3: 추적 시스템 (F4) — Done
추천 종목의 성과를 추적하여 시스템 자체의 실적을 측정.

| 구성 요소 | 설명 |
|-----------|------|
| 추천 기록 | `recommendations` 테이블에 저장 |
| 성과 추적 | Phase 변화, 가격 변동 자동 업데이트 |
| 리포트 반영 | 주간 에이전트에 과거 성과 주입 |

### Layer 4: 애널리스트 토론 시스템 (F6) — Done
4명의 전문 애널리스트가 3라운드 토론을 수행.

| 구성 요소 | 설명 |
|-----------|------|
| 4 애널리스트 | 매크로, 테크, 지정학, 심리 |
| 3라운드 | 독립분석 → 교차검증(반박 필수) → 모더레이터 종합 |
| 뉴스 사전 수집 | Brave Search로 애널리스트별 뉴스 주입 |
| 모멘텀 데이터 | 5일/20일 가격 변화율 → 고점 피로감 감지 |
| 환각 방지 | 프롬프트 레벨 수치 정확성 규칙 |

### Layer 5: Thesis 추적 + 학습 루프 (F6 확장) — Done
예측을 기록하고 검증하는 핵심 피드백 시스템.

| 구성 요소 | 설명 |
|-----------|------|
| Thesis 저장 | 토론에서 추출된 전망 → `theses` 테이블 |
| LLM 자동 검증 | 시장 데이터 기반 CONFIRMED/INVALIDATED/HOLD 판정 |
| 만료 자동화 | timeframe 초과 thesis → EXPIRED |
| 학습 승격 | 3회+ 적중 패턴 → `agent_learnings` 승격 (이항 검정 유의성 필터) |
| 기억 주입 | 학습 + 검증 결과를 토론 프롬프트에 반영 |
| 편향 감지 | bull-bias 80% 초과 경고, 정량 검증 비율 모니터링 |

### Layer 6: 펀더멘탈 검증 (F7) — Done
기술적 신호를 펀더멘탈로 교차 검증.

| 구성 요소 | 설명 |
|-----------|------|
| SEPA 스코어링 | Minervini 기준 (EPS/매출 YoY, 가속, 마진) |
| 등급 | S → A → B → C → F |
| LLM 리포트 | S등급 종목 개별 분석 리포트 |

### Layer 7: 시그널 검증 + 품질 관리 — Done
초입 포착 도구의 유효성을 데이터로 검증하고, 시스템의 자기확증편향을 감지하는 체계.

| 구성 요소 | 설명 |
|-----------|------|
| Phase 1 후기 검증 | 전환율 41.9% → **유효** (PR #61) |
| RS 상승 초기 검증 | 전환율 20.3% → **부분 유효**, 섹터 동반 상승 시 24.2% |
| 펀더멘탈 가속 검증 | Phase 1 교집합 30.6% → **교집합이 핵심** |
| QA 에이전트 | GitHub Actions + 파일 저장 + 프롬프트 강화 (PR #62) |
| 편향 감지 | 이항 검정(p < 0.05 + Cohen's h ≥ 0.3) + bull-bias 경고 (PR #63) |
| 펀더멘탈 확장 | SEPA 스코어링 전체 종목 확장 + DB 저장 (PR #67) |

**핵심 발견:** 섹터 RS 동반 상승이 가장 유의미한 필터. 단독 도구보다 교집합이 강력.

### Layer 8: 주간 리포트 재설계 — Done
일간 스냅샷의 반복이 아닌, 주간 관점의 방향성과 속도를 분석하는 리포트 체계. (PR #69)

| 구성 요소 | 설명 |
|-----------|------|
| 도구 주간 모드 | getIndexReturns, getMarketBreadth, getLeadingSectors, readRecommendationPerformance에 weekly 모드 추가 |
| 프롬프트 차별화 | 방향·속도 해석 중심, 일간 수치 사용 금지, 섹터 RS 동반 상승 필터 필수 |
| Phase 이탈 집계 | 이번 주 추천 종목 Phase 변경 건 추적 |

### Phase N: 서사 레이어 + 검증 강화 — "왜 이 종목인가"의 구조적 근거

**배경:** PLTR(23), NVDA(24), MU/LITE(25) 같은 알파 종목은 기술적 신호 이전에 구조적 서사(수요 폭발 → 공급 체인 → 병목)가 먼저 존재했다. 이 수요-공급-병목 체인은 AI뿐 아니라 90년대 인터넷, 스마트폰, 에너지 전환 등 **모든 메가트렌드에서 반복된 범용 구조**다.

**원칙:** 기존 파이프라인(Phase + RS + 펀더멘탈)은 그대로 유지하고, 서사 레이어의 출력과 **교집합**을 취한다. "교집합이 최강 필터"라는 기존 발견의 연장선.

**설계 원칙:** "데이터가 필요하다고 말할 때까지 테이블을 만들지 마라." 별도 `narrative_chains` 테이블은 만들지 않고, thesis 카테고리 분리로 먼저 운영한 후 데이터 패턴을 보고 결정.

> 원본 RFC: `docs/RFC-narrative-layer.md`

### Phase B: 데이터 차별화 — "남들이 안 보는 것을 본다"

현재 데이터는 전부 공개 데이터(가격, RS, Phase). 이것만으로는 엣지가 없다.

**실질 순서:** 1인 운영 프로젝트이므로 병렬 진행은 비현실적. N-2 대기 기간(~2주) 중 B-1 착수 → N-2 착수 기준 충족 시 B-1 일시 중단 → N-2 완료 후 B-1 재개.

#### B-1. 섹터 간 자금 흐름 추적
**현재 상태:** 섹터별 RS 순위만 존재. 자금이 어디서 빠져서 어디로 가는지 흐름이 안 보임.
**목표:** RS 변화율 + Phase 2 비율 변화의 **교차 분석**으로 로테이션 방향 감지.

```
예시 출력:
"Technology RS -5.2 (3일) / Energy RS +8.1 (3일) → Tech→Energy 로테이션 진행 중"
"Healthcare Phase2 비율 18% → 24% (5일) → 자금 유입 초기 신호"
```

**난이도:** 낮음 (기존 데이터로 파생 가능) | **효과:** 중간

#### B-2. 거래량 이상 감지 강화
**현재 상태:** Phase 2 진입 시 거래량 확인(volumeConfirmed) 존재. 하지만 섹터 레벨의 거래량 이상은 안 봄.
**목표:** 섹터/산업 레벨에서 "평소 대비 비정상적 거래량 급증" 감지 → 스마트머니 유입 초기 신호.

**난이도:** 중간 | **효과:** 높음

#### B-3. 뉴스 DB 내재화 (선택적)
**현재 상태:** 뉴스는 토론 직전에 Brave Search로 수집 후 휘발.
**목표:** 뉴스를 DB에 저장하여 (1) 과거 뉴스-시장 반응 패턴 축적 (2) 학습 데이터로 활용.

**판단:** 현재 우선순위 낮음. 뉴스 내재화는 학습 루프가 충분히 성숙한 후에 의미가 있다.

**난이도:** 높음 | **효과:** 장기적으로 높음

### Phase C: 출력 품질 고도화 — "읽는 사람이 바로 활용한다"

#### C-1. 리포트 후처리 검증
**현재 상태:** LLM 출력을 그대로 발송. 환각 수치가 포함될 수 있음.
**목표:** 리포트 내 수치를 ETL 데이터와 대조하여 **검증 불가 수치 자동 플래깅 또는 제거**.

```
리포트: "27년 만의 최대 괴리" → ETL에 역사적 데이터 없음 → 자동 제거 또는 [미검증] 태그
리포트: "NVDA RS 95" → ETL 확인: RS 95 → 통과
```

**난이도:** 중간 | **효과:** 높음 — 신뢰도 직접 향상

#### C-2. 시각화 (장기)
**현재 상태:** 텍스트 리포트만 발송.
**목표:** 섹터 RS 히트맵, Phase 분포 차트, 모멘텀 스코어보드 등.

**난이도:** 높음 | **효과:** 높음 (UX)

## 솔직한 리스크

1. **LLM 환각은 완전히 제거 불가** — 프롬프트 + 후처리로 줄일 수 있지만, 0은 불가능. 수치가 포함된 모든 문장을 의심해야 함.
2. **학습 데이터 축적에 시간 필요** — 유의미한 패턴이 나오려면 최소 3개월, 이상적으로 6개월의 데이터 필요.
3. **시장 레짐 변화** — 축적된 패턴이 시장 국면 변화 시 무효화될 수 있음. 학습 만료 메커니즘(현재 6개월)이 이를 완화.
4. **비용** — Claude API ~$15/월 (프롬프트 캐싱 적용 후, N-1 확장 후 재측정 필요). 학습 루프가 복잡해지면 증가.
5. **단일 운영자 의존** — 파이프라인(ETL, 토론, 리포트)은 자동이지만, 에러 대응/thesis 리뷰/학습 승격 판단은 사람 개입 필요. 1~2주 손을 못 대면: (a) 자동 파이프라인은 계속 동작, (b) 에러 시 Discord 알림으로 감지, (c) thesis 검증·승격은 자동이므로 지연 없음, (d) 다만 QA 리포트 리뷰와 시스템 방향 판단은 멈춤. 현재 자동화 범위로 단기 부재는 허용 가능하나, 월간 점검은 필수.

## 결론

이 시스템의 진짜 가치는 LLM의 "분석력"이 아니라 **데이터 파이프라인 + 예측 추적 + 학습 축적의 복리 효과**에 있다.

6개월 후 이 시스템이 가진 것:
- 수백 건의 검증된/무효화된 thesis
- 어떤 시장 조건에서 어떤 판단이 맞았는지의 통계
- 그 통계가 반영된 에이전트 프롬프트

이것은 어떤 뉴스레터도, 유튜버도, 단독 LLM 호출도 가질 수 없는 것이다.
핵심은 **시간이라는 해자**. 지금 시작해서 매일 쌓는 것 자체가 경쟁 우위다.
