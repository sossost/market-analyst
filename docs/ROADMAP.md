# Market Analyst — Roadmap

**최종 업데이트:** 2026-04-14

---

## 최종 목적지

**시간이 지날수록 똑똑해지는 시장 분석 시스템.**

구체적으로:
1. 매일 미국 시장 전체를 스캔하여 **상승 초입(Phase 2) 주도섹터/주도주를 남들보다 먼저 포착**
2. 예측을 기록하고 검증하여 **어떤 조건에서 맞았고 틀렸는지 패턴을 축적**
3. 축적된 패턴이 다음 분석에 반영되어 **시스템의 판단 정확도가 시간과 함께 향상**

이것은 "LLM이 시장을 이해하는 시스템"이 아니다.
**데이터 파이프라인 + 패턴 축적 + 규율 있는 추적**이 만드는 복리 효과를 추구하는 시스템이다.

### 이 시스템이 아닌 것
- 트레이딩 시그널 생성기 (목표가/손절가 없음)
- 단기 매매 도구 (구조적 변화의 초기 신호가 목표)
- LLM의 "직관"에 의존하는 시스템 (데이터가 말하게 한다)

### 성공의 정의

| 기준 | 목표 | 측정 방법 |
|------|------|----------|
| **thesis 적중률 (1번 KPI)** | 50%+ | CONFIRMED / (CONFIRMED + INVALIDATED) — thesis 20건+ 검증 완료 후 |
| **포착 선행성 (2번 KPI)** | 컨센서스보다 1-2주 먼저 | 관심종목 등록일 vs 실제 상승 시작일 비교 (관심종목 10건+ 결과 확정 후) |
| 학습 패턴 축적 | 적중률 60%+ 학습 10개 이상 | `agent_learnings` 테이블 (6개월 후) |
| 리포트 활용도 | 실제 투자 판단에 월 1건+ 활용 | **월 1회 수신자 피드백** — 리포트별 유용성 3점 척도(활용함/참고함/무시함) + 자유 코멘트. Discord DM 또는 간단 폼으로 수집 |

## 현재까지 완료한 것

### Layer 1: 데이터 인프라 (F1) — Done
시스템의 기반. 매일 미국 전체 종목을 스캔하고 구조화된 데이터를 생산.

|-----------|------|

| Sector/Industry RS | 상대 강도 점수 + 가속도 + 브레드스 (Phase 2 비율) + 업종 오분류 보정(`symbol_industry_overrides` COALESCE 패턴, #722) |
| 펀더멘탈 데이터 | 분기 재무제표 + 비율 (FMP API) |
| 돌파/노이즈 신호 | 기술적 시그널 필터링 (breakout_signals, noise_signals) |
| 지수 가격 ETL | S&P 500, NASDAQ, DOW, Russell 2000, VIX 일간 가격 (FMP API → `index_prices` 테이블) — #420 |
| 종목 촉매 데이터 | stock_news(종목 뉴스), earning_calendar(실적 일정), eps_surprises(어닝 서프라이즈) — Phase 2/관심종목 대상, ETL Phase 3.9 병렬 실행 — #456 |
| 신용 지표 ETL | FRED API — HY OAS(`BAMLH0A0HYM2`), CCC Spread(`BAMLH0A3HYC`), BBB Spread(`BAMLC0A4CBBB`), Financial Stress Index(`STLFSI4`) 일간 수집 + 180일 z-score 이상 감지 → 토론 컨텍스트 전 에이전트 주입 — #748 |
| 추천 자동화 | scan-recommendation-candidates — Phase 2 전수 스캔 → 8개 게이트(+실패 패턴 필터 #799) 통과 종목을 ETL Phase 3.8에서 자동 저장. 에이전트 호출 누락으로 인한 0건 리스크 제거 (#547) |
| 시장 브레드스 스냅샷 | `market_breadth_daily` — Phase 분포·A/D ratio·52주 신고가/신저가·VIX(종가+일중고점)·공포탐욕 지수를 일별 스냅샷으로 축적. 대시보드 heavy join 제거 (#588, #590). `breadth_score`(5개 지표 252일 퍼센타일 가중합산 0~100) + `divergence_signal`(가격-브레드스 다이버전스 자동 감지) 컬럼 추가 (#600). `phase1_to2_count_1d`·`phase2_to3_count_1d`(당일 Phase 전환 카운트) 컬럼 추가 + 일간 리포트 HTML stat-chip 표시 (#657) |

**파이프라인:** 맥미니 launchd 단독, 화~토 KST 07:00 (= UTC 22:00 Mon-Fri), ~50분 소요.

### Layer 7: 시그널 검증 + 품질 관리 — Done

| 구성 요소 | 설명 |
|-----------|------|
| Phase 1 후기 검증 | 전환율 41.9% → **유효** (PR #61) |
| RS 상승 초기 검증 | 전환율 20.3% → **부분 유효**, 섹터 동반 상승 시 24.2% |
| 펀더멘탈 가속 검증 | Phase 1 교집합 30.6% → **교집합이 핵심** |

### Phase N: 서사 레이어 + 검증 강화 — "왜 이 종목인가"의 구조적 근거

**배경:** PLTR(23), NVDA(24), MU/LITE(25) 같은 알파 종목은 기술적 신호 이전에 구조적 서사(수요 폭발 → 공급 체인 → 병목)가 먼저 존재했다. 이 수요-공급-병목 체인은 AI뿐 아니라 90년대 인터넷, 스마트폰, 에너지 전환 등 **모든 메가트렌드에서 반복된 범용 구조**다.

**원칙:** 기존 파이프라인(Phase + RS + 펀더멘탈)은 그대로 유지하고, 서사 레이어의 출력과 **교집합**을 취한다. "교집합이 최강 필터"라는 기존 발견의 연장선.

**설계 원칙:** "데이터가 필요하다고 말할 때까지 테이블을 만들지 마라." 별도 `narrative_chains` 테이블은 만들지 않고, thesis 카테고리 분리로 먼저 운영한 후 데이터 패턴을 보고 결정.

### Phase D: 분석 정확도 고도화 — "판단의 질을 측정하고 보정한다"

**배경:** 시스템이 "맞다/틀리다"를 축적하는 건 되지만, "얼마나 확신했고 그 확신이 정확했는가"를 모른다. 확신도 90%로 내린 판단이 실제 60%만 맞다면, 시스템의 신뢰 구간이 왜곡된 채 운영되는 것. 레짐별 성과 차이도 보이지 않으면 "이 시스템이 어떤 장에서 강한지"를 모른 채 운영하게 된다.

**원칙:** 새 데이터를 수집하는 게 아니라, 이미 쌓인 데이터(theses, market_regimes, debate sessions)를 교차 분석하여 시스템의 자기 인식을 높인다.

| # | 항목 | 설명 | 난이도 | 우선순위 |
|---|------|------|--------|----------|
| D-1 | 신뢰도 캘리브레이션 | thesis confidence vs 실제 적중률 보정 곡선. "90% 확신이 실제 90% 맞는가?" 검증 | 중간 | P2 |
| D-2 | 레짐별 성과 분석 | `market_regimes` × `theses` 조인 → 레짐별 적중률 분리. "이 시스템은 어떤 장에서 강한가?" | 낮음 | P2 |
| D-3 | 소수 의견 보존 | 토론 종합(synthesis) 시 다수와 다른 소수 의견을 별도 기록 → 사후 "소수가 맞았던 비율" 추적 | 낮음 | P2 |
| D-4 | 다양성 강제 (악마의 변호인) | 분석 다양성 점수 측정 → 임계값 미달 시 반대 입장 강제 주입 | 중간 | P3 |

**착수 기준:**
- D-1, D-2: thesis 검증 데이터 20건+ 축적 시 (현재 ~4건, N-2 대기와 병렬 가능)

- D-4: D-3 운영 후 소수 의견 데이터 충분히 쌓인 후

**N-2와의 관계:** N-2는 "학습 주입이 도움이 되는가"를 검증하고, Phase D는 "판단 자체의 품질을 보정"한다. 상호 보완적이며 독립 실행 가능.

### Phase P: 모델 포트폴리오 — "검증된 인사이트로 알파를 만든다"

**배경:** 시스템의 최종 가치는 "인사이트가 맞았는가"를 넘어 "인사이트가 실제 알파로 전환되는가"에 있다. 2단계(thesis 적중률 + 포착 선행성)가 안정적으로 검증된 후, 그 인사이트를 기계적 규칙으로 모델 포트폴리오에 반영하여 벤치마크 대비 초과수익을 측정한다.

**핵심 원칙:**
- LLM이 매매 판단을 하지 않는다 — 인사이트를 **기계적 규칙**으로 포트폴리오에 반영
- 매크로 레짐별 자산배분은 **통계적 백테스트**로 도출 (LLM 직관 아님)
- 포트폴리오 성과는 인사이트의 질을 간접 측정하는 도구

**진화 경로:**
```
1단계: 추천종목 수익률 판정 (폐기 — LLM 산출물을 수익률로 판정하면 불공정)
2단계: 관심종목 + thesis 적중률 + 포착 선행성 (현재)
3단계: 검증된 인사이트 → 모델 포트폴리오 → 알파 측정 (Phase P)
```

**착수 기준 (전제 조건):**
- Thesis 적중률 50%+ 안정적 유지 (3개월 연속)
- 포착 선행성 컨센서스 대비 1주+ 선행 확인
- 관심종목 20건+ 결과 확정

| # | 항목 | 설명 | 난이도 |

**이것이 Phase 1과 다른 이유:**
Phase 1은 "LLM이 추천한 종목의 수익률"을 직접 판정했다. Phase P는 "검증된 인사이트를 기계적 규칙으로 반영한 포트폴리오"의 성과를 측정한다. LLM은 인사이트를 생산할 뿐, 매매 판단은 규칙이 한다.

## 솔직한 리스크

1. **LLM 환각은 완전히 제거 불가** — 프롬프트 + 후처리로 줄일 수 있지만, 0은 불가능. 수치가 포함된 모든 문장을 의심해야 함.
2. **학습 데이터 축적에 시간 필요** — 유의미한 패턴이 나오려면 최소 3개월, 이상적으로 6개월의 데이터 필요.
3. **시장 레짐 변화** — 축적된 패턴이 시장 국면 변화 시 무효화될 수 있음. 학습 만료 메커니즘(현재 6개월)이 이를 완화.
4. **비용** — Claude API ~$15/월 + OpenAI/Gemini API 과금 (멀티 모델 토론 도입). 학습 루프가 복잡해지면 증가.
5. **단일 운영자 의존** — 파이프라인(ETL, 토론, 리포트)은 자동이지만, 에러 대응/thesis 리뷰/학습 승격 판단은 사람 개입 필요. 1~2주 손을 못 대면: (a) 자동 파이프라인은 계속 동작, (b) 에러 시 Discord 알림으로 감지, (c) thesis 검증·승격은 자동이므로 지연 없음, (d) 다만 QA 리포트 리뷰와 시스템 방향 판단은 멈춤. 현재 자동화 범위로 단기 부재는 허용 가능하나, 월간 점검은 필수.

## 결론

이 시스템의 진짜 가치는 LLM의 "분석력"이 아니라 **데이터 파이프라인 + 예측 추적 + 학습 축적의 복리 효과**에 있다.

6개월 후 이 시스템이 가진 것:
- 수백 건의 검증된/무효화된 thesis
- 어떤 시장 조건에서 어떤 판단이 맞았는지의 통계
- 그 통계가 반영된 에이전트 프롬프트

이것은 어떤 뉴스레터도, 유튜버도, 단독 LLM 호출도 가질 수 없는 것이다.
핵심은 **시간이라는 해자**. 지금 시작해서 매일 쌓는 것 자체가 경쟁 우위다.

그리고 그 축적이 충분히 검증되면, **모델 포트폴리오로 실제 알파를 측정하는 것**이 자연스러운 다음 단계다.
인사이트의 질 → 포트폴리오 성과 → 시스템의 최종 가치 증명. 이것이 이 프로젝트의 완전한 여정이다.
