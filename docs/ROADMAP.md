# Market Analyst — Roadmap

**최종 업데이트:** 2026-03-06
**작성:** 비서실장

---

## 최종 목적지

**시간이 지날수록 똑똑해지는 시장 분석 시스템.**

구체적으로:
1. 매일 미국 시장 전체를 스캔하여 **상승 초입(Phase 2) 주도섹터/주도주를 남들보다 먼저 포착**
2. 예측을 기록하고 검증하여 **어떤 조건에서 맞았고 틀렸는지 패턴을 축적**
3. 축적된 패턴이 다음 분석에 반영되어 **시스템의 판단 정확도가 시간과 함께 향상**

이것은 "LLM이 시장을 이해하는 시스템"이 아니다.
**데이터 파이프라인 + 패턴 축적 + 규율 있는 추적**이 만드는 복리 효과를 추구하는 시스템이다.

### 이 시스템이 아닌 것
- 트레이딩 시그널 생성기 (목표가/손절가 없음)
- 단기 매매 도구 (구조적 변화의 초기 신호가 목표)
- LLM의 "직관"에 의존하는 시스템 (데이터가 말하게 한다)

### 성공의 정의
- Phase 2 진입 섹터/종목을 **컨센서스보다 1-2주 먼저** 포착
- 6개월 후 축적된 패턴 중 **적중률 60% 이상**인 학습이 10개 이상
- 리포트 수신자가 **실제 투자 판단에 활용**하는 수준의 품질

시스템의 기반. 매일 미국 전체 종목을 스캔하고 구조화된 데이터를 생산.

|-----------|------|
| Stock Phases | Weinstein Stage Analysis — 8개 조건으로 Phase 1~4 판별 |
| Sector/Industry RS | 상대 강도 점수 + 가속도 + 브레드스 (Phase 2 비율) |
| 펀더멘탈 데이터 | 분기 재무제표 + 비율 (FMP API) |
| 돌파/노이즈 신호 | 기술적 시그널 필터링 |

**파이프라인:** GitHub Actions, 매일 UTC 00:00, ~50분 소요.

| 리뷰 에이전트 | 리포트 품질 검증 파이프라인 |
| 딜리버리 | Discord + Gist, 일간/주간 스케줄 |

### Layer 3: 추적 시스템 (F4) — Done
추천 종목의 성과를 추적하여 시스템 자체의 실적을 측정.

| 구성 요소 | 설명 |
|-----------|------|
| 추천 기록 | `recommendations` 테이블에 저장 |

| 리포트 반영 | 주간 에이전트에 과거 성과 주입 |

### Layer 4: 토론 시스템 (F6) — Done, 품질 개선 진행 중

| 구성 요소 | 설명 |

| 뉴스 사전 수집 | Brave Search로 페르소나별 뉴스 주입 |
| 모멘텀 데이터 | 5일/20일 가격 변화율 → 고점 피로감 감지 |

### Layer 5: Thesis 추적 + 학습 루프 (F6 확장) — Done (기초)
예측을 기록하고 검증하는 핵심 피드백 시스템.

| 구성 요소 | 설명 |
|-----------|------|
| Thesis 저장 | 토론에서 추출된 전망 → `theses` 테이블 |

| 기억 주입 | 학습 + 검증 결과를 토론 프롬프트에 반영 |

## 앞으로 해야 할 것

### Phase A: 학습 루프 완성 — "에이전트가 경험에서 배우게 만든다"

현재 학습 루프의 한계: 학습이 **메모리 텍스트**로만 주입되고, 에이전트 프롬프트에 직접 영향을 주지 않는다. 실질적으로 "배운다"고 하기 어렵다.

#### A-1. 검증된 학습 → 에이전트 프롬프트 자동 주입
**현재 상태:** `memoryLoader.ts`가 학습을 불러오지만, 일반 텍스트 덩어리로 삽입.
**목표:** 페르소나별로 관련 학습만 필터링해서, 분석 규칙 바로 아래에 "과거 검증된 패턴" 섹션으로 구조화 삽입.

```
예시 — macro-economist 프롬프트에 자동 삽입:

## 검증된 패턴 (시스템 자동 생성, 과거 데이터 기반)
- Energy RS > 70 + Phase2 비율 > 50% → 2주 내 섹터 조정 (적중률 75%, 4회 관측)
- VIX 20 → 30 급등 시, Russell 2000 가장 큰 타격 (적중률 67%, 3회 관측)
```

**구현 범위:** `memoryLoader.ts` 수정 + 페르소나별 필터링 로직
**난이도:** 낮음 | **효과:** 중간

#### A-2. Few-shot 사례 주입 — 과거 성공/실패 분석
**현재 상태:** 검증 결과가 "적중/빗나감" 한 줄로만 전달됨. 당시 시장 상황이나 분석 맥락이 없음.
**목표:** 과거 토론 → 결과를 **시장 조건 + 분석 + 실제 결과** 3종 세트로 저장. 유사 조건 발생 시 few-shot으로 주입.

```
예시:

## 과거 유사 상황 참고
[2026-02-15] VIX 22, Energy RS 65, Phase2 비율 28%
- 매크로: "Energy 추가 상승 여력" → 실제: 2주 후 Energy RS 73까지 상승 (적중)
- 테크: "AI 반도체 반등 임박" → 실제: 추가 5% 하락 후 반등 (타이밍 오류)
- 교훈: Energy 모멘텀은 맞췄으나 테크 바닥 타이밍은 1주 빨랐음
```

**필요한 것:**
- 토론 결과 + 당시 시장 스냅샷을 `debate_sessions` 테이블에 저장
- 토론 시작 시 현재 시장 조건으로 유사 세션 검색 (SQL 기반, 벡터 불필요)
- 상위 2-3개 유사 세션을 few-shot으로 주입

**난이도:** 중간 | **효과:** 높음 — 이게 가장 큰 품질 점프를 만들 것

#### A-3. 학습 자동 정제
**현재 상태:** `promote-learnings`가 3회+ 적중 시 학습 승격. 하지만 학습 내용이 thesis 원문 그대로라 추상화/정제가 안 됨.
**목표:** LLM으로 여러 적중 thesis를 종합하여 **일반화된 원칙**으로 정제.

```
raw: "NVDA RS 95에서 2주 내 조정" + "AAOI RS 98에서 10일 내 조정" + "AMCR RS 99에서 1주 내 조정"
→ 정제: "RS 95+ 종목은 1-2주 내 단기 조정 확률 높음 (3/3 관측)"
```

**난이도:** 중간 | **효과:** 중간

---

### Phase B: 데이터 차별화 — "남들이 안 보는 것을 본다"

현재 데이터는 전부 공개 데이터(가격, RS, Phase). 이것만으로는 엣지가 없다.

#### B-1. 섹터 간 자금 흐름 추적
**현재 상태:** 섹터별 RS 순위만 존재. 자금이 어디서 빠져서 어디로 가는지 흐름이 안 보임.
**목표:** RS 변화율 + Phase 2 비율 변화의 **교차 분석**으로 로테이션 방향 감지.

```
예시 출력:
"Technology RS -5.2 (3일) / Energy RS +8.1 (3일) → Tech→Energy 로테이션 진행 중"
"Healthcare Phase2 비율 18% → 24% (5일) → 자금 유입 초기 신호"
```

**난이도:** 낮음 (기존 데이터로 파생 가능) | **효과:** 중간

#### B-2. 거래량 이상 감지 강화
**현재 상태:** Phase 2 진입 시 거래량 확인(volumeConfirmed) 존재. 하지만 섹터 레벨의 거래량 이상은 안 봄.
**목표:** 섹터/산업 레벨에서 "평소 대비 비정상적 거래량 급증" 감지 → 스마트머니 유입 초기 신호.

**난이도:** 중간 | **효과:** 높음

#### B-3. 뉴스 DB 내재화 (선택적)
**현재 상태:** 뉴스는 토론 직전에 Brave Search로 수집 후 휘발.
**목표:** 뉴스를 DB에 저장하여 (1) 과거 뉴스-시장 반응 패턴 축적 (2) 학습 데이터로 활용.

**판단:** 현재 우선순위 낮음. A단계(학습 루프)와 B-1/B-2가 먼저. 뉴스 내재화는 학습 루프가 충분히 성숙한 후에 의미가 있다.

**난이도:** 높음 | **효과:** 장기적으로 높음

---

### Phase C: 출력 품질 고도화 — "읽는 사람이 바로 활용한다"

#### C-1. 리포트 후처리 검증
**현재 상태:** LLM 출력을 그대로 발송. 환각 수치가 포함될 수 있음.
**목표:** 리포트 내 수치를 ETL 데이터와 대조하여 **검증 불가 수치 자동 플래깅 또는 제거**.

```
리포트: "27년 만의 최대 괴리" → ETL에 역사적 데이터 없음 → 자동 제거 또는 [미검증] 태그
리포트: "NVDA RS 95" → ETL 확인: RS 95 → 통과
```

**난이도:** 중간 | **효과:** 높음 — 신뢰도 직접 향상

#### C-2. 시각화 (장기)
**현재 상태:** 텍스트 리포트만 발송.
**목표:** 섹터 RS 히트맵, Phase 분포 차트, 모멘텀 스코어보드 등.

**난이도:** 높음 | **효과:** 높음 (UX)

## 솔직한 리스크

1. **LLM 환각은 완전히 제거 불가** — 프롬프트 + 후처리로 줄일 수 있지만, 0은 불가능. 수치가 포함된 모든 문장을 의심해야 함.
2. **학습 데이터 축적에 시간 필요** — 유의미한 패턴이 나오려면 최소 3개월, 이상적으로 6개월의 데이터 필요.
3. **시장 레짐 변화** — 축적된 패턴이 시장 국면 변화 시 무효화될 수 있음. 학습 만료 메커니즘(현재 6개월)이 이를 완화.
4. **비용** — Claude API 호출 비용이 토론 1회당 ~$0.5-1.0. 학습 루프가 복잡해지면 비용 증가.

---

## 결론

이 시스템의 진짜 가치는 LLM의 "분석력"이 아니라 **데이터 파이프라인 + 예측 추적 + 학습 축적의 복리 효과**에 있다.

6개월 후 이 시스템이 가진 것:
- 수백 건의 검증된/무효화된 thesis
- 어떤 시장 조건에서 어떤 판단이 맞았는지의 통계
- 그 통계가 반영된 에이전트 프롬프트

이것은 어떤 뉴스레터도, 유튜버도, 단독 LLM 호출도 가질 수 없는 것이다.
핵심은 **시간이라는 해자**. 지금 시작해서 매일 쌓는 것 자체가 경쟁 우위다.
