## 골 정렬

**ALIGNED** — 잘못된 데이터가 포함된 브리핑은 Phase 2 포착 판단 자체를 오염시킨다. WTI $79 vs $90 차이는 에너지 섹터 온도 판단을 완전히 반전시킬 수 있다. 수신자 신뢰도 파괴는 시스템 전체 가치를 훼손한다.

단, 세 가지 제안 중 원인과 해결 복잡도가 다르므로 Phase 분할이 필요하다.

## 문제

브리핑 생성 시 LLM이 실시간 조회 도구를 호출하지 않는 지표(WTI, 금, 달러 인덱스 등 원자재/거시 지표)에 대해 학습 데이터 내 수치를 그대로 사용한다. 이 수치는 실제 시장과 수일~수개월 차이가 날 수 있다.

부가 문제로, Discord 요약 메시지와 Gist 상세 리포트 간 수치가 LLM이 두 번 별도로 생성하는 과정에서 불일치할 수 있다. 또한 매일 받는 수신자에게 용어 설명을 반복 출력하는 것은 노이즈다.

## Before → After

### 문제 1: 원자재/거시 지표 시점 불명확 및 LLM 환각
- **Before**: LLM이 WTI, 금, DXY 등 실시간 데이터 없이 브리핑을 작성 가능. 수치를 환각하거나 오래된 데이터를 인용함.
- **After (Phase 1)**: 시스템 프롬프트에 "실시간 조회 불가 지표는 수치를 직접 언급하지 말 것" 명시. 출처 없는 거시 수치 사용 차단.
- **After (Phase 2)**: 원자재/거시 데이터 실시간 조회 도구 추가 (WTI, 금, DXY 등 Yahoo Finance 동일 패턴). 브리핑에 데이터 기준 시점 자동 삽입.

### 문제 2: Discord 요약 vs Gist 상세 수치 불일치
- **Before**: LLM이 message(Discord)와 markdownContent(Gist)를 동일 컨텍스트에서 생성하지만, 수치를 두 번 직접 작성하므로 오타/반올림 차이 가능.
- **After (Phase 1)**: 시스템 프롬프트에 "message와 markdownContent의 수치는 반드시 일치해야 함" 명시. 불일치 시 자체 검토 절차 추가.
- **After (Phase 2)**: 검토 가능하다면 send_discord_report 직전 수치 일관성 검증 레이어 추가.

### 문제 3: 용어 설명 반복
- **Before**: MD 파일 매 하단에 Phase/RS/MA150/A/D ratio 용어 설명 전체 반복.
- **After (Phase 1)**: 시스템 프롬프트에 조건 추가 — "정기 구독자(매일 발송)이므로 기존 용어 설명 섹션을 생략하라." 단, 용어 첫 등장 시 괄호 약어 설명은 유지.

## 변경 사항

### Phase 1 — 프롬프트 레벨 (코드 변경 최소, 즉시 효과)

**파일: `src/agent/systemPrompt.ts`**

1. **원자재/거시 수치 환각 차단 규칙** (Bull-Bias 가드레일 섹션 이후 추가):
   ```
   ## 데이터 시점 규칙
   - **실시간 조회 불가 지표(WTI, 금, 은, DXY, 원화환율 등)**: 수치를 직접 브리핑에 언급하지 마세요.
     "WTI $XX.XX" 형태로 쓸 수 있는 것은 도구로 조회된 데이터만입니다.
     수치가 없으면 "원자재 동향은 당일 시장 데이터 미수집으로 생략"으로 처리하세요.
   - **수치 출처**: 브리핑에 인용된 모든 수치는 이 세션에서 도구로 조회한 결과여야 합니다.
     학습 데이터 내 수치를 추론하거나 기억에서 가져오는 행위는 엄격히 금지됩니다.
   ```

2. **Discord vs Gist 수치 일관성 규칙** (규칙 섹션에 추가):
   ```
   - **message와 markdownContent 수치 일치**: send_discord_report 호출 전,
     message(Discord 요약)와 markdownContent(Gist 상세)에 등장하는 동일 지표의 수치가
     완전히 일치하는지 자체 검토하세요. 불일치가 있으면 markdownContent 기준으로 통일하세요.
   ```

3. **용어 설명 섹션 조건화** (용어 설명 섹션 지시 수정):
   ```
   ## 용어 설명 (정기 발송 제외)
   일간 브리핑은 정기 구독자에게 매일 발송됩니다. MD 파일 하단의 "용어 설명" 섹션을 생략하세요.
   용어가 처음 등장할 때 괄호 내 약어 설명(예: RS(상대강도))은 유지합니다.
   ```

### Phase 2 — 도구 레벨 (별도 이슈, 본 기획서 범위 외)

다음은 Phase 1 완료 후 효과 검증 기간(1~2주) 이후에 착수한다:
- `getCommodityPrices` 도구 신설: Yahoo Finance API로 CL=F(WTI), GC=F(금), DX-Y.NYB(DXY) 실시간 조회. `getIndexReturns.ts` 패턴 그대로 적용.
- 조회 결과에 `data_as_of` (거래소 마지막 거래 타임스탬프) 포함.
- 브리핑 메시지 헤더에 `데이터 기준: YYYY-MM-DD HH:MM ET` 자동 삽입.

## 리스크

- **근본 원인은 도구 부재**: 프롬프트 규칙은 LLM이 "수치를 모르는 것"처럼 행동하도록 강제하지만, 모델이 자신 있게 잘못된 수치를 쓸 가능성을 완전히 차단하지는 못한다. Phase 2 도구 신설이 근본 해결책이다.
- **용어 설명 제거 시 신규 수신자 혼란**: 현재 수신자가 정기 구독자뿐이면 문제 없으나, 신규 유입 시 첫 수신 브리핑에서 용어가 불친절해질 수 있다. 단, 이슈에서 "매일 받는 수신자에게 노이즈"로 명시했으므로 제거가 의도된 방향.
- **Phase 1 효과 측정 어려움**: 프롬프트 규칙 추가 후 LLM이 실제로 원자재 수치 환각을 안 하는지 검증하려면 해당 수치가 포함된 브리핑이 나와야 한다. 다음 WTI 언급 브리핑에서 수동 확인 필요.

## 의사결정 필요
