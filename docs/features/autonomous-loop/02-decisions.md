# Decisions: 자율 개선 루프

## Decision 1: Layer 1 (기계적 시그널)부터 시작

**결정**: LLM 루프보다 기계적 시그널 자동 튜닝을 먼저 구현한다.

**근거**:
- 비용 $0, 측정 명확, 오염 없음
- 백테스트 결과가 이미 있어서 즉시 파라미터 조정 가능
- LLM 루프는 데이터 축적이 필요 (최소 2-3개월)
- 기계적 루프가 작동해야 LLM 루프의 부가가치를 측정할 수 있음

**대안**: LLM 토론 품질 개선 먼저 → 기각. 측정 기준이 없는 상태에서 품질 개선은 방향 없는 노력.

## Decision 2: 파라미터 조정 방식 — 그리드 서치 + 롤링 윈도우

**결정**: 매주 최근 20거래일 데이터로 파라미터 조합별 성과를 재계산하고, 베스트 파라미터를 자동 채택.

**근거**:
- 시장 국면이 바뀌면 최적 파라미터도 바뀜 (고정하면 과적합)
- 20거래일 = 약 1개월, 최소한의 통계적 의미 있는 기간
- 그리드 서치는 단순하지만 파라미터 수가 적어서 충분

**대안**: 강화학습 기반 → 기각. 데이터 부족, 복잡도 대비 이점 불명확.

## Decision 3: CEO 리포트 채널 분리

**결정**: 일간 투자 리포트와 별도로 #ceo-brief 채널에 주간 시스템 리포트 발송.

**근거**:
- 투자 리포트: "뭘 사야 하나" (매일)
- CEO 리포트: "시스템이 어떻게 돌아가고 있나" (매주)
- 성격이 다르므로 채널 분리가 합리적

## Decision 4: signal_log 테이블은 recommendations 대체가 아닌 병렬

**결정**: 기존 recommendations 테이블은 유지하고, signal_log는 기계적 시그널 전용으로 별도 운영.

**근거**:
- recommendations는 LLM 에이전트가 생성 (주관적 추천)
- signal_log는 기계적 조건 충족 시 자동 생성 (객관적 시그널)
- 둘을 비교하면 "LLM이 가치를 더하는가?"를 직접 측정 가능

## Decision 5: 매니저 메타 판단은 세션 시작 시 자동 로드

**결정**: chief-of-staff.md를 MEMORY.md처럼 매 세션 자동 로드하여, 축적된 판단 기준이 항상 반영되도록 한다.

**근거**:
- 지금은 실수할 때만 기록하고, 다음 세션에서 참조하지 않는 경우가 있음
- 자동 로드하면 매 세션 시작 시 "지난 번에 이런 실수를 했고, 이런 기준을 세웠다"가 컨텍스트에 포함
