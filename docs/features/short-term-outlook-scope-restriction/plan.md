# Plan: short_term_outlook thesis 범위 제한

## 문제 정의

`short_term_outlook` 카테고리 적중률 39.1% (9 CONFIRMED / 14 INVALIDATED / 3 EXPIRED).
랜덤(50%)보다 낮아, 시스템이 이 카테고리 예측의 반대로 행동하는 것이 더 유리한 수준.

**핵심 원인**: 시장 심리/변동성의 **타이밍**을 예측하는 thesis가 실패.
- 공포탐욕지수 바닥 형성 → 25 이상 회복 예측 → 실제 10.2로 추가 악화
- VIX 20 이하 진입 예측 → 실제 31.05로 상승
- 상승 전환 비율 35% 돌파 예측 → 30% 저항선 재이탈

**피해**: `agent_learnings`의 caution 항목 3건 중 2건이 short_term_outlook 기반 → 학습 루프 오염

## Before → After

| 항목 | Before | After |
|------|--------|-------|
| short_term_outlook 범위 | 제한 없음 (타이밍 예측 포함) | 타이밍 예측 금지, 구조적 전환·조건부 방향성만 허용 |
| 프롬프트 가이드 | 카테고리 분류 기준 1줄 | 허용/금지 패턴 구체적 명시 |
| 전체 thesis 품질 | 57.1% (오염 포함) | short_term_outlook 품질 개선 → 전체 적중률 상승 기대 |

## 변경 사항

### 1. thesis 생성 프롬프트 수정 (`src/debate/round3-synthesis.ts`)

**카테고리 분류 기준** 섹션에 short_term_outlook 범위 제한 규칙 추가:

- **금지 패턴** (구체적 예시 포함):
  - 특정 날짜/기한 내 수치 도달 예측 (예: "30일 내 VIX 20 하회", "2주 내 공포탐욕지수 25 회복")
  - 심리/변동성 지표의 절대 수준 예측 (예: "VIX 20 이하 진입", "공포탐욕지수 25 이상 회복")
  - 단기 가격 반등/하락 타이밍 예측

- **허용 패턴**:
  - 구조적 전환 방향성 (예: "risk-off → risk-on 전환 초기 신호")
  - 조건부 형식 (예: "VIX가 25 이하로 안정 시 기술주 반등 가능")
  - 레짐 전환 감지 (예: "EARLY_BEAR에서 EARLY_BULL 전환 조건 형성 중")

### 2. 테스트 추가 (`src/debate/__tests__/thesis-category-filter.test.ts`)

프롬프트 규칙은 LLM 행동 가이드이므로 코드 레벨 강제가 어렵지만,
`buildSynthesisPrompt`가 새 규칙을 포함하는지 검증하는 테스트 추가.

## 골 정렬

- **ALIGNED**: thesis 적중률(현재 57.1%)을 KPI 목표(50%+) 이상으로 유지/개선.
  short_term_outlook의 저품질 예측을 줄여 학습 루프 오염을 방지하고,
  structural_narrative(90.9%)와 sector_rotation(62.5%)의 성과를 보존.

## 무효 판정

- **해당 없음**: 카테고리를 폐지하지 않고 범위만 제한하므로 기존 데이터/시스템 파괴 없음.
  레짐 전환 감지 등 유용한 short_term_outlook 신호는 유지.

## 작업 계획

1. `round3-synthesis.ts` 프롬프트 수정 — 카테고리 분류 기준에 범위 제한 추가
2. 테스트 추가 — 프롬프트에 새 규칙 문자열이 포함되는지 검증
3. 기존 테스트 통과 확인

## 리스크

- **LLM 우회 가능성**: 프롬프트 규칙은 강제가 아닌 가이드. 다만 구체적 금지 패턴 예시를 포함하여 우회 확률을 최소화.
- **기존 ACTIVE thesis**: 현재 ACTIVE 15건 중 타이밍 예측에 해당하는 것은 자연 만료되므로 별도 처리 불필요. 신규 생성부터 적용.
