# 투자 브리핑 이벤트 캘린더 가독성 개선

## 선행 맥락

- `debate-markdown-render` 피처(완료): 프론트엔드 아카이브 대시보드에 `react-markdown + remark-gfm` 적용. 마크다운 테이블이 대시보드에서는 완전히 렌더링됨.
- `round3-synthesis.ts` 프롬프트의 섹션 4(주도섹터/주도주)는 이미 마크다운 테이블 형식으로 지정되어 있어 LLM이 테이블을 출력함 — 동일 패턴을 섹션 7에도 적용하면 됨.

## 골 정렬

SUPPORT — 리포트 가독성 향상 → CEO가 이벤트 캘린더를 빠르게 파악 → Phase 2 포착 의사결정 보조. 직접 분석 기능은 아니지만 분석 결과 소비 품질 개선.

## 문제

`round3-synthesis.ts`의 브리핑 프롬프트 섹션 7 "이벤트 캘린더"가 출력 형식을 `YYYY-MM-DD | 이벤트명 | 영향` 파이프 구분자 텍스트로만 지정하고 있어, LLM이 줄바꿈 없이 한 줄로 이어쓰는 경우가 발생한다.

## Before → After

**Before**: 섹션 7 프롬프트 지시 (164~167번째 줄):
```
### 7. 이벤트 캘린더
- 다음 1~2주 내 주요 이벤트를 **날짜 오름차순**으로 정렬
- 형식: YYYY-MM-DD | 이벤트명 | 위 분석에 미치는 영향 (1줄)
```
LLM 출력 예시: `2026-03-17 | FOMC 의사록 공개 | 금리 정책 변화 신호 시... 2026-03-20 | 이란-이스라엘 외교 회담 예정 | ...`

**After**: 섹션 7 프롬프트를 마크다운 테이블 형식으로 지정:
```
### 7. 이벤트 캘린더
다음 1~2주 내 주요 이벤트를 **날짜 오름차순**으로 마크다운 테이블 형식으로 작성하세요:

| 날짜 | 이벤트 | 위 분석에 미치는 영향 |
|------|--------|---------------------|
| YYYY-MM-DD | 이벤트명 | 영향 1줄 |
```
GitHub Gist(상세 리포트)와 프론트엔드 대시보드 양쪽에서 테이블로 렌더링됨.

## Discord 렌더링 분석

리포트 발송 경로는 두 채널로 분리되어 있다:

| 채널 | 내용 | 마크다운 테이블 지원 |
|------|------|-------------------|
| Discord 본문 (summary) | 핵심 요약 2000자 이내 — 이벤트 캘린더 섹션 불포함 | 미지원(무관) |
| GitHub Gist -> Discord 링크 (report) | cleanReport 전문 — 섹션 7 포함 | 완전 지원 |
| MD 파일 첨부 fallback | Gist 실패 시 .md 파일로 전송 | 파일 뷰어에서 지원 |

**결론**: 이벤트 캘린더는 Discord 본문이 아닌 Gist/MD 파일로 전달됨. 마크다운 테이블로 변경해도 Discord 렌더링 문제 없음.

## 리스크

- **LLM 지시 불이행 가능성 낮음**: 섹션 4에 동일 패턴(테이블 예시 포함 지시)이 이미 적용되어 있고 LLM이 이를 잘 따름. 동일 패턴 적용이므로 리스크 낮음.
- **기존 파싱 로직 영향 없음**: `extractDebateOutput`은 JSON 코드블록만 파싱함. 마크다운 테이블 형식 변경은 파싱에 무관.
- **테스트 업데이트 필요**: `buildSynthesisPrompt` 출력을 검증하는 스냅샷 테스트가 있으면 문자열 변경으로 실패할 수 있음 → 스냅샷 업데이트 필요.
