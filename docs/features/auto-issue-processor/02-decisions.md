# 의사결정 기록 — 자율 이슈 처리 시스템

## Decision 1: Phase 1부터 자율 실행 포함

**날짜**: 2026-03-10
**상태**: accepted

### 컨텍스트
Phase 1을 트리아지 전용으로 할지, 자율 실행까지 포함할지.

### 결정
**Phase 1부터 트리아지 + 자율 실행 + PR 생성까지 포함.**

### 근거
- CEO 판단: "어차피 PR 리뷰하는 데 시간 많이 안 쓰고, PR 리젝트하면 끝"
- 트리아지만 하면 CEO가 여전히 "착수" 지시를 해야 해서 자동화 효과 반감
- PR이 안전망 역할을 하므로 코드베이스 오염 리스크 없음 (머지 전까지)
- 2주 검증 기간 없이 바로 가치 제공

---

## Decision 2: 트리아지 제거 — 이슈 → 바로 실행

**날짜**: 2026-03-10
**상태**: accepted (supersedes: LLM 트리아지)

### 컨텍스트
초기에는 LLM(Claude) 기반 트리아지로 이슈를 "자율 처리 가능"과 "CEO 판단 필요"로 분류했으나, CEO가 PR 리뷰에서 최종 판단하므로 트리아지 단계가 불필요한 오버헤드.

### 결정
**트리아지 레이어 제거. 미처리 이슈를 바로 실행하여 PR 생성.**

### 근거
- CEO가 PR 리뷰에서 최종 판단 → 사전 분류 불필요
- 트리아지 LLM 비용 절감 (~1K tokens/건)
- `auto:queued`, `auto:needs-ceo` 라벨 제거로 상태 머신 단순화
- 브랜치 타입은 이슈 타이틀 접두사(fix:, feat: 등)에서 추출, 기본값 fix

---

## Decision 3: 하루 4회 스케줄 — StartCalendarInterval 나열

**날짜**: 2026-03-10
**상태**: accepted (수정: 10회 → 4회)

### 결정
launchd `StartCalendarInterval`에 10/12/14/16시 × 5요일을 개별 dict로 명시.

### 근거
- 주 2~3건 이슈 볼륨에 10회/일은 과도 → 4회로 충분
- 기존 plist 패턴과 일관성 유지
- `StartInterval`은 근무 시간 범위 제한 불가
- 20개 항목 (4시간 × 5요일)

---

## Decision 4: 중복 방지 — GitHub 라벨 기반

**날짜**: 2026-03-10
**상태**: accepted

### 결정
`auto:` 접두사 라벨 유무로 처리 여부 판단. 로컬 파일이나 DB 불사용.

### 근거
- GitHub에 상태가 있으면 어느 환경에서든 동기화 없이 판단 가능
- CEO가 GitHub UI에서 현황을 직관적으로 파악 가능
- 맥미니 이관 시에도 상태 소실 없음

---

## Decision 5: 에스컬레이션 — 이슈 코멘트 + 라벨만

**날짜**: 2026-03-10
**상태**: accepted

### 결정
Discord/Slack 알림 없이 GitHub 이슈 코멘트 + 라벨만. Phase 2에서 알림 추가 검토.

### 근거
- CEO가 GitHub을 직접 확인하고 있어 충분
- 알림 인프라는 추가 설정 필요 — Phase 1 범위 밖

---

## Decision 6: 판단 보수성 — 전부 실행, PR이 안전망

**날짜**: 2026-03-10
**상태**: accepted (수정: Decision 2 트리아지 제거 반영)

### 결정
모든 미처리 이슈를 바로 실행. PR 리뷰가 최종 안전망.

### 근거
- 트리아지 제거(Decision 2)로 분류 단계 없음
- 잘못 처리해도 CEO가 PR 리젝트하면 끝
- 실행 실패 시 라벨 제거 + 실패 사유 코멘트 → 다음 사이클 재시도 없음

---

## Decision 7: 실행 도구 — Claude Code CLI

**날짜**: 2026-03-10
**상태**: accepted

### 컨텍스트
이슈를 실제로 구현하는 방법.

### 결정
**Claude Code CLI (`claude`)를 subprocess로 실행.** 이슈 본문 + 프로젝트 컨텍스트를 프롬프트로 전달.

### 근거
- Claude Code CLI가 이미 맥미니에 설치되어 있음
- CLAUDE.md 기반 프로젝트 컨텍스트 자동 로드
- 브랜치 생성, 코드 작성, 커밋, PR 생성까지 한 번에 가능
- 기존 미션 프로토콜(mission-planner → 구현 → review → PR)을 그대로 활용

## Decision 8: 1사이클 최대 2건 제한

**날짜**: 2026-03-10
**상태**: accepted

### 결정
한 번의 스케줄 실행에서 최대 2건만 처리.

### 근거
- Claude Code CLI 실행은 건당 수 분~수십 분 소요
- 한 사이클에 너무 많은 이슈를 처리하면 다음 사이클과 겹칠 수 있음
- 비용 제어: 일일 최대 ~8건 (4사이클 × 2건, 실제로는 1~3건 예상)
