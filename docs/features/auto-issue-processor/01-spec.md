# 자율 이슈 처리 시스템

## 선행 맥락

- **launchd 인프라 완비** (PR #64): 맥미니에서 etl-daily, debate-daily, agent-weekly, qa-weekly 등 6개 잡 운영 중. 신규 잡 추가는 plist 1개 + setup-launchd.sh 배열 추가만으로 완료.
- **머지 권한 (ORGANIZATION.md)**: CEO 지시 없는 머지 절대 금지. 자율 처리 시스템도 PR 생성까지만. 머지는 CEO.

## 골 정렬

**SUPPORT** — 직접 주도주 포착 기능은 아니나, 매니저가 이슈 처리에 소비하는 인지 부하를 제거하여 전략적 판단에 집중할 수 있게 한다.

## 문제

CEO가 열린 이슈를 하나씩 매니저에게 전달해야 처리가 시작된다. CEO의 가용 시간에 이슈 처리 속도가 종속됨.

## Before → After

**Before**
- CEO가 이슈를 발견 → 매니저에게 전달 → 매니저가 처리 시작
- 이슈 처리 속도가 CEO의 가용 시간에 종속

**After**
- 매니저가 평일 10:00, 12:00, 14:00, 16:00 KST 하루 4회 이슈를 자율 점검
- 자율 처리 가능한 이슈: 분석 → 구현 → 리뷰 → PR 생성까지 자동
- CEO 판단이 필요한 이슈: 이슈 코멘트로 에스컬레이션
- CEO는 PR만 확인하고 머지 결정만 한다

## 범위

### Phase 1 (MVP) — 이번 구현 대상

**핵심**: 미처리 이슈 → 바로 Claude Code CLI 실행 → PR 생성.

CEO가 PR 리뷰에서 최종 판단하므로 LLM 트리아지 단계 불필요. 이슈가 올라오면 바로 실행.

**포함:**
1. 스케줄러 스크립트 (`scripts/cron/issue-processor.sh`)
2. launchd plist (`com.market-analyst.issue-processor.plist`) — 평일 10/12/14/16시 KST, 하루 4회
3. 자율 실행 파이프라인 (`src/issue-processor/executeIssue.ts`)
   - 미처리 이슈 → Claude Code CLI로 처리
   - 이슈 타이틀에서 브랜치 타입 추출 (fix:, feat:, refactor:, chore:)
   - 브랜치 생성 → 구현 → PR 생성

**제외 (Phase 2):**
- Discord/Slack 알림
- 처리 결과 DB 저장 및 통계
- 일일 처리 건수 제한 자동 조절

## 라벨 상태 머신

```
[신규 이슈] (auto: 라벨 없음)
    │
    ▼ 파이프라인 실행
[auto:in-progress]
    │
    ├── 성공 → [auto:done] + PR 링크 코멘트
    └── 실패 → 라벨 제거 + 실패 사유 코멘트
```

**라벨 정의:**

| 라벨 | 의미 | 자동/수동 |
|------|------|---------|
| `auto:in-progress` | 파이프라인 실행 중 | 자동 |
| `auto:done` | PR 생성 완료 | 자동 |

**중복 방지**: 이미 `auto:` 접두사 라벨이 붙은 이슈는 스킵.

## 실행 파이프라인

미처리 이슈를 바로 처리하는 순서:

```
1. 이슈 타이틀에서 브랜치 타입 추출 (fix:/feat:/refactor:/chore:, 기본값 fix)
2. 브랜치 생성 ({타입}/issue-{번호})
3. Claude Code CLI 실행 — 이슈 본문을 프롬프트로 전달
4. 변경사항 커밋 + 푸시
5. PR 생성 (gh pr create)
6. 이슈에 PR 링크 코멘트
7. 라벨: auto:in-progress → auto:done
```

**실패 시:**
- 라벨: `auto:in-progress` 제거
- 이슈에 실패 사유 코멘트
- 다음 이슈로 넘어감 (격리)

**1사이클 제한**: 한 번의 스케줄 실행에서 최대 2건 처리 (비용 + 시간 제한)

## 실패/에러 처리

| 시나리오 | 처리 |
|---------|------|
| GitHub API 인증 실패 | 로그 기록 + 다음 사이클 재시도 |
| 이슈 조회 실패 | 로그 기록 + 종료 |
| 라벨 부착 실패 | 로그 기록 + 코멘트만 달고 계속 |
| Claude Code CLI 실패 | `auto:in-progress` 제거 + 실패 사유 코멘트 |
| PR 생성 실패 | `auto:in-progress` 제거 + 실패 사유 코멘트 |
| 스크립트 예외 | stderr 로그 기록 + 비정상 종료 |

**원칙**: 에러가 나도 다른 이슈 처리에 영향을 주지 않는다. 이슈별 try-catch 격리.

## 보안 고려사항

| 항목 | 처리 방법 |
|------|---------|
| GitHub PAT | 맥미니 환경 변수 (`GITHUB_TOKEN`). `.env` 파일 `.gitignore` 처리. |
| Claude API 키 | 기존 `ANTHROPIC_API_KEY` 환경 변수 재사용. |
| 이슈 내용 신뢰 | 이슈 본문을 셸 명령에 직접 삽입하지 않는다. JSON API 응답만 파싱. |
