# Spec: 주간 리포트 전면 재설계

GitHub Issue: #69

## 문제

주간 리포트가 일간 리포트와 사실상 동일하다. 도구가 일간 스냅샷만 반환하고, 프롬프트도 주간 맥락을 요구하지 않아서 에이전트가 주간 고유의 분석을 물리적으로 수행할 수 없다.

## 목적

주간 리포트를 "이번 주 구조가 어떻게 바뀌었는가"를 보여주는 고유 출력물로 재설계한다. 일간 노이즈를 걸러내고, Phase 2 초입 포착 신뢰도를 높이는 주간 맥락을 제공한다.

## 주간 리포트만의 고유 가치 (일간에서는 불가능한 것)

1. **섹터 로테이션 방향 판단** — 5일 연속 RS 상승 섹터는 실질적 자금 유입 증거. 하루치로는 노이즈와 신호 구분 불가.
2. **Phase 2 진입 추세의 가속/감속** — 주간 Phase 2 비율 추이선(18%->19%->21%->22%->23%)으로 모멘텀 판단.
3. **구조적 변화 vs 군중 심리 구분** — 섹터 RS 4주 추세와 이번 주 상위 섹터 일치 = 구조적, 불일치 = 일회성.
4. **전주 판단 사후 검증** — 지난주 리포트의 전망이 실제로 어떻게 됐는지 복기. 시스템 신뢰도 축적.
5. **추천 코호트 성과 + 오판 명시** — 주간 승률/평균 수익률 집계, Phase 이탈 종목 공개.
6. **thesis 충돌 대응** — 충돌 인지만 하지 않고, 비중 조절/조건부 추천/thesis 재검토 트리거.

## 요구사항

### 도구 확장 (기존 도구에 `mode: "weekly"` 파라미터 추가)

#### A. `getIndexReturns`
- `mode: "weekly"` 시 주 5거래일 데이터 반환
- 주간 누적 수익률 (`weeklyChangePercent`)
- 금요일 종가가 주간 고점/저점 중 어디에 위치하는지 (`closePosition: "near_high" | "near_low" | "mid"`)
- 전주 대비 방향 연속성 (`weekDirection: "up" | "down"`, `consecutiveWeeks: number`)

#### B. `getMarketBreadth`
- `mode: "weekly"` 시 직전 5거래일 Phase 2 비율 추이 배열 반환
- 주간 Phase 1->2 전환 종목 수 합계 (`phase1to2Transitions`)
- A/D ratio 주간 누적 방향
- 52주 신고가 종목수 주간 합산 + 전주 대비 변화

#### C. `getLeadingSectors`
- `mode: "weekly"` 시 전주 대비 RS 순위 변동 포함
- 각 섹터에 `prevWeekRank`, `rankChange`, `prevWeekAvgRs` 추가
- 신규 진입/이탈 섹터 리스트 (`newEntrants`, `exits`)
- 2주 연속 상위 3 유지 섹터 별도 표시

#### D. `readRecommendationPerformance`
- `period: "this_week"` 시 주간 성과 집계
- 이번 주 신규 추천 수, 종료 건 수, 주간 승률, 평균 PnL
- Phase 이탈 종목 리스트 (이번 주 Phase 2에서 다른 Phase로 변경된 것)
- 오판 케이스 명시 (Phase 2 추천 후 1주 내 Phase 1/3으로 복귀)

### 프롬프트 재설계 (`buildWeeklySystemPrompt`)

#### 워크플로우 변경
- 1~3단계에 `mode: "weekly"` 호출 명시
- "일간 등락률" 문구 전면 제거
- 섹터 로테이션 해석 지시: "전주 대비 순위 5+ 상승 섹터 = 신규 자금 유입 초기 신호"
- 구조적/일회성 구분 규칙: "섹터 RS 4주 추세와 이번 주 상위 섹터 일치 = 구조적, 불일치 = 일회성"
- thesis 충돌 대응: 에이전트가 "이 thesis는 이번 주 데이터로 흔들렸다" 판단 허용

#### 리포트 포맷 변경

메시지 1 — 주간 시장 흐름:
```
S&P 500: X,XXX | 주간 +X.XX% (전주 +X.XX%)
Phase 2 추이: XX% -> XX% -> XX% -> XX% -> XX%
섹터 로테이션: SectorA (상승 +5위) / SectorB (하락 -3위)
```

메시지 2 — 주도주 (기존 유지, 선정 기준 강화):
- S/A등급 우선, B등급만으로 구성 금지
- 섹터 RS 동반 상승 필터 강제
- 2주 연속 RS 상위 유지 종목 우선

메시지 3 — 예비군 (기존 유지):
- 이번 주 신규 Phase 2 전환 x 섹터 RS 동반 상승 교집합 하이라이트

메시지 4 — 인사이트 + 성과:
- 전주 판단 사후 검증 (지난주 주목한 섹터/종목이 실제로 어떻게 됐나)
- 주간 추천 성과 집계 (승률, 평균 PnL, 오판 케이스)
- thesis 충돌 경고 (ACTIVE thesis와 이번 주 추천의 방향 불일치)

### 주도주 선정 기준 강화
- 섹터 RS 동반 상승 필터 강제 (#61 검증 결과)
- S/A등급 우선 배치, B등급만으로 주도주 구성 금지
- 펀더멘탈 가속 음수(악화 추세) 필터링
- 단일 섹터 편중 경고 (동일 섹터 3개 이상이면 명시적 경고)

## 스코프

**In scope:**
- 기존 4개 도구에 `mode: "weekly"` / `period` 파라미터 추가
- `buildWeeklySystemPrompt` 워크플로우 + 포맷 재설계
- 주도주 선정 기준 강화
- 도구별 단위 테스트

**Out of scope:**
- 신규 도구 추가 (기존 확장으로 충분)
- DB 스키마 변경 (기존 테이블로 전부 가능)
- 일간 리포트 변경 (영향 없음)
- 외부 매크로 지표 추가 (수익률 곡선, DXY 등 — 현재 데이터 소스 없음)

## 기술 제약

- 모든 파라미터에 기본값 `"daily"` / `"all"` 설정 → 기존 일간 에이전트 하위 호환 보장
- 주간 기준일은 직전 금요일로 고정 (에이전트 날짜 계산 실수 방지)
- Yahoo Finance 휴장일 대응: 실제 close 배열 길이 체크, 2개 미만이면 일간 fallback
- `sector_rs_daily` 7일 전 데이터 부재 시 가장 가까운 이전 주 날짜로 대체

## 완료 기준

- [ ] 주간 리포트가 주간 누적 지수 등락률을 표시한다
- [ ] Phase 2 비율 5일 추이선이 포함된다
- [ ] 섹터 RS 전주 대비 순위 변동이 표시된다
- [ ] 추천 성과 주간 집계(승률, 평균 PnL)가 포함된다
- [ ] S/A등급 종목이 우선 배치되고 B등급만 리포트는 불가능하다
- [ ] thesis 충돌 시 명시적 경고가 포함된다
- [ ] 기존 일간 에이전트가 영향 없이 동작한다
- [ ] 도구별 단위 테스트 통과, 커버리지 80% 유지
