# Plan: 전일 리포트 컨텍스트 정확성 강화

> Closes #521 — [일간 QA] 2026-03-30 전일 특이종목 누락 허위 서술, 브레드스 데이터 모순

## 문제 정의

3/30 일간 리포트에서 3건의 팩트 일관성 결함 발생:

| # | 결함 | 원인 | 심각도 |
|---|------|------|--------|
| 1 | "전일 특이종목 없음" 허위 서술 | `<previous-report>` 컨텍스트에 종목 목록이 있음에도 LLM이 무시/할루시네이션 | HIGH |
| 2 | 브레드스 데이터 "조회 불가"라 서술하면서 동일 데이터를 테이블로 제공 | LLM의 자기모순 — 도구 결과와 서술 간 정합성 미검증 | MEDIUM |
| 3 | EEIQ 동일 카탈리스트를 반대 방향으로 해석 (전일 -17.3% → 금일 +17.3%) | 전일 등락률이 컨텍스트에 없어 연속성 서술 불가 | MEDIUM |

## Before → After

| 항목 | Before | After |
|------|--------|-------|
| 전일 특이종목 컨텍스트 | 종목 목록만 나열 (Phase, RS, 섹터) | **총 N건 카운트 + 등락률 포함 + "없음 서술 금지" 명시적 가드** |
| 브레드스 정합성 | 검증 규칙 없음 | **"데이터 제공 시 조회 불가 서술 금지" 프롬프트 규칙 추가** |
| 연속 등장 종목 | 전일 등장 여부만 태그 ([강세]/[약세]) | **전일 등락률 포함 + 연속 등장 시 맥락 비교 필수 규칙** |
| fullContent 파싱 fallback | reportedSymbols 비어있으면 "없음" | **fullContent에서 추출한 종목으로 fallback** |

## 변경 사항

### 1. `src/lib/previousReportContext.ts`

**a. `extractStockReturns()` 신규 함수**
- fullContent에서 각 티커의 등락률(±XX.X%) 추출
- 패턴: `TICKER.*?([+-]\d+\.?\d*%)` 또는 `([-+]\d+\.?\d*%).*?TICKER`

**b. `formatPreviousReportContext()` 강화**
- 특이종목 카운트 요약 라인 추가: `⚠️ 직전 리포트 특이종목 총 N건 (강세 X, 약세 Y) — "전일 특이종목 없음" 서술 금지`
- 종목 라인에 등락률 포함: `- EEIQ (Phase 2, RS 65, Tech) [약세] | 전일 -17.3%`

**c. fullContent 기반 fallback**
- `reportedSymbols`가 비어있으나 fullContent에서 종목 추출 가능 시, fullContent 기반 목록 생성

### 2. `src/agent/systemPrompt.ts`

3개 프롬프트 규칙 추가:

**a. 브레드스 데이터 정합성 자기검증**
> 도구 결과에서 데이터를 정상 수신하여 리포트 테이블/수치에 사용했으면, 같은 리포트에서 해당 데이터를 "조회 불가"로 서술 금지

**b. 연속 등장 종목 맥락 필수**
> 동일 종목이 `<previous-report>`와 오늘 `get_unusual_stocks` 모두에 등장 시, 전일 등락률·방향을 먼저 언급 후 오늘 변화 서술. 동일 카탈리스트 반대 해석 시 방향 전환 근거 필수

**c. "전일 특이종목 없음" 명시적 금지 강화**
> `<previous-report>` 내 특이종목이 1건 이상이면 "전일 특이종목 없음/데이터 없음" 서술 절대 금지. 총 N건 카운트가 명시되어 있으므로 반드시 인용

### 3. 테스트

- `extractStockReturns()` 유닛 테스트
- 강화된 포맷 (카운트 요약, 등락률 포함) 검증
- fullContent fallback 검증

## 리스크

- **fullContent 등락률 파싱 정확도**: 리포트 포맷이 바뀌면 regex가 깨질 수 있음 → fail-open (추출 실패 시 등락률 없이 기존 형식 유지)
- **프롬프트 길이 증가**: 규칙 3개 추가 (~300 토큰) → 전체 프롬프트 대비 미미

## 골 정렬

- **ALIGNED** — 리포트 팩트 일관성은 프로젝트 핵심 품질 지표. QA 총점 28/40 → 32+ 목표.
