# Plan: 토론 에이전트 실패 시 자동 재시도 + Discord 알림

**이슈:** #741
**날짜:** 2026-04-11

## 문제 정의

`etl-daily.sh` L106에서 토론 에이전트(`run-debate-agent.ts`)가 실패하면 재시도 없이 즉시 `exit 1`.
일시적 인증 에러(Opus CLI "Not logged in") 한 번으로 일간 리포트 전체가 스킵된다.

토론 없이 리포트를 발행하면 품질 보장이 불가하므로, 폴백 리포트는 부적절.
→ 재시도로 일시적 장애를 극복하되, 최종 실패 시 CEO에게 알린다.

## 골 정렬

- **판정: SUPPORT** — 토론 에이전트는 thesis 검증 + 시장 레짐 판단의 핵심 단계.
  토론 실패 → 일간 리포트 미발행 → Phase 2 초입 포착 시그널이 CEO에게 전달되지 않음.
  직접적인 포착 기능은 아니지만, 포착 결과의 전달 안정성을 보장하는 인프라 개선.
- **무효 판정: 해당 없음** — 운영 안정성 개선이므로 무효 조건 없음.

## Before → After

| 항목 | Before | After |
|------|--------|-------|
| 토론 실패 시 | 즉시 exit 1, 리포트 스킵 | 3분 대기 후 재시도 (최대 2회), 총 3회 시도 |
| 실패 알림 | `send_error` 1회 호출 | 최종 실패 시 `send_error`로 3회 시도 실패 + 수동 개입 필요 알림 |
| 로깅 | 실패 로그 1줄 | 각 시도마다 몇 번째 시도인지 + 실패 시 재시도 대기 로그 |

## 변경 사항

### 수정 파일: `scripts/cron/etl-daily.sh` (1개)

**L104~129 토론 실행 블록을 재시도 루프로 교체:**

1. `DEBATE_MAX_RETRIES=2`, `DEBATE_RETRY_DELAY=180` 상수 선언
2. for 루프로 최대 3회 시도 (초회 + 재시도 2회)
3. 각 시도마다 로그 출력 ("토론 에이전트 시도 N/3")
4. 성공 시 루프 탈출 → 기존 promote-learnings + QA 흐름 진행
5. 실패 시:
   - 마지막 시도가 아니면: 대기 시간 로그 + `sleep 180`
   - 마지막 시도면: `send_error`로 "3회 시도 실패, 수동 재실행 필요" 알림 + exit 1
6. 기존 `send_error` 함수 재사용 — 새 Discord 패턴 추가 불필요

  단, 주간 에이전트(Sonnet)가 동시간대 정상이었으므로 일시적 에러에는 합리적.
- **파이프라인 지연**: 최악 시 +6분(3분×2회). 일간 파이프라인 50분 기준으로 12% 증가. 수용 가능.
