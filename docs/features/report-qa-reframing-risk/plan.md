# Plan: 전일 동일 수치 재프레이밍 + 리스크 섹션 누락 해소

## 문제 정의

3/24 리포트 QA에서 4가지 품질 결함 감지 (GitHub #429):

1. **전일 동일 수치 재프레이밍**: Financial Services 84건이 전일과 동일한데 "구조적 변화"로 재프레이밍
2. **리스크 섹션 누락**: 약세 시장(VIX 27, 공포탐욕 14.5)에서 독립 리스크 섹션이 "향후 관전 포인트"에 흡수
3. **해석 격상 근거 미명시**: 전일 "지속성 검증 필요" → 금일 "대규모 전환 신호"로 추가 근거 없이 격상
4. **예비군 교체 이유 미서술**: 직전 예비군 탈락/신규 진입 이유 부재

## 골 정렬

**SUPPORT** — Phase 2 주도섹터/주도주 초입 포착 목표 직접 지원은 아니나, 리포트 품질 개선은 시그널 정확도에 직결. 잘못된 재프레이밍은 false positive 시그널을 유발.

## 무효 판정

해당 없음 — LLM 백테스트/시뮬레이션 아님. 프롬프트 가드레일 + 정적 검증기 강화.

## Before → After

| 항목 | Before | After |
|------|--------|-------|
| 동일 수치 재프레이밍 | 검증 없음 | 프롬프트 가드레일 + 직전 리포트 주요 수치 컨텍스트 |
| 리스크 섹션 | 키워드 빈도만 체크 | 약세 시장(VIX 25+)에서 독립 리스크 섹션 필수 검증 |
| 해석 격상 | 프롬프트 수준 안내만 | 프롬프트에 격상 시 근거 필수 규칙 추가 |
| 예비군 교체 | 검증 없음 | 프롬프트에 교체 사유 서술 규칙 + 직전 예비군 컨텍스트 |

## 변경 사항

### 1. `src/agent/systemPrompt.ts` — 프롬프트 가드레일 추가
- **동일 수치 가드레일**: 전일과 동일한 수치는 "유지"로 서술. "변화"로 프레이밍하려면 전일 대비 달라진 근거 필수.
- **리스크 섹션 독립 규칙**: 시장 온도가 약세(VIX 25+ 또는 공포탐욕 25 이하)일 때 리스크/경고 요인을 독립 섹션으로 유지.
- **해석 격상 근거 규칙**: 전일 유보적이던 해석을 확정적으로 바꿀 때 추가 데이터 근거 명시 필수.
- **예비군 교체 사유 규칙**: 직전 예비군 종목 탈락/신규 진입 시 1줄 이상 사유 서술 필수.

### 2. `src/lib/previousReportContext.ts` — 직전 리포트 주요 수치 추가
- Phase 2 비율, 주도 섹터 RS 건수 등 핵심 수치를 컨텍스트에 포함하여 동일 수치 비교 근거 제공.
- 직전 예비군 종목 목록 추가.

### 3. `src/lib/reportValidator.ts` — 정적 검증기 강화
- **M. 독립 리스크 섹션 검증**: 약세 키워드(VIX 25+, 공포, 약세)가 시장 온도에 있으면 독립 리스크/경고 섹션 존재 확인.
- **N. 예비군 교체 사유 검증**: "전일 대비" 섹션에서 예비군 변경 언급 시 사유 키워드 존재 확인.

### 4. `src/lib/__tests__/reportValidator.test.ts` — 테스트 추가
- 독립 리스크 섹션 검증 테스트
- 예비군 교체 사유 검증 테스트

## 작업 계획
