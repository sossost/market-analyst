# Plan: strategic-review 동일 주제 이슈 반복 생성 방지

> Lite 트랙 — 프롬프트 파일 1개만 수정하는 단순 fix

## 문제 정의

strategic-review가 매일 실행되면서 동일 근본 원인에 대해 반복 이슈를 생성한다.
대표 사례: sentiment 에이전트 적중률 문제가 3주간 9건으로 반복 (#361~#687).

**근본 원인:** 현재 프롬프트에 OPEN 이슈 체크만 있고, CLOSED 이슈에 대한 쿨다운이 없다.
PR 머지 → 이슈 CLOSED → 다음날 동일 수치로 동일 이슈 재생성하는 루프.

## Before → After

### Before
- OPEN 이슈만 확인 → CLOSED되면 다음날 동일 이슈 재생성
- 90일 윈도우 지표는 PR 머지 후에도 수치 변화 미미 → 무한 루프

### After
- 3단계 중복 체크: OPEN → 최근 30일 CLOSED → PR 머지 쿨다운
- PR 머지 후 14일 쿨다운 / PR 없이 닫힌 경우 7일 쿨다운
- 쿨다운 예외: 수치 5%p 이상 악화 또는 새 근본 원인 발견 시 허용
- 스킵 시 strategic-briefing.md에 쿨다운 상태 기록

## 의사결정

| 항목 | 결정 | 근거 |
|------|------|------|
| 쿨다운 기간 | PR 있으면 14일, 없으면 7일 | 90일 윈도우 특성상 14일이면 충분한 관찰 기간. 30일은 과도 |
| 수치 악화 임계값 | 5%p | 너무 작으면 노이즈 반응, 너무 크면 진짜 악화 놓침 |
| CLOSED 이슈 조회 범위 | 30일 | 쿨다운 최대 14일의 2배. 충분한 버퍼 |
