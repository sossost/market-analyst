# Report Layouts -- 리포트 레이아웃 및 데이터 구조

> 최종 갱신: 2026-04-18
> 일간/주간 HTML 리포트의 섹션 구성, 각 섹션에 들어가는 데이터, UI 컴포넌트, 색상 체계를 정리한다.

---

## 공통 설계 원칙

1. **데이터와 해석 분리** -- 정량 데이터는 프로그래밍으로 렌더링, LLM은 해석 텍스트만 작성
2. **Self-contained HTML** -- 외부 CSS/JS 의존 없음. `<style>` 태그 내장
3. **XSS 방어** -- 모든 동적 값 `escapeHtml()` 처리, `javascript:` 링크 차단
4. **한국식 색상** -- 상승=빨강(#cf222e), 하락=파랑(#0969da)
5. **반응형** -- 모바일(600px 이하) 대응, CSS Grid `auto-fit`

---

## 색상 체계

### CSS 변수

```
--up:      #cf222e    상승 (빨강)
--down:    #0969da    하락 (파랑)
--phase2:  #8250df    Phase 2 (보라) [일간]
           #1a7f37    Phase 2 (초록) [주간]
--yellow:  #9a6700    Phase 3 / 경고
--orange:  #bc4c00    주의
--purple:  #8250df    강조
```

### Phase 색상

| Phase | 배지 배경 | 텍스트 | Phase Bar |
|-------|----------|--------|-----------|
| P1 | #eef1f4 (연회색) | text-muted | #d0d7de |
| P2 | #f0e6ff (연보라) | --phase2 | #8250df (보라) |
| P3 | #fff8c5 (연황색) | --yellow | #eac054 |
| P4 | #ffebe9 (연빨강) | --down | #cf222e |

### 시장 온도 배지

| 온도 | 배경 | 텍스트 |
|------|------|--------|
| bullish | #ffebe9 (연빨강) | --up |
| neutral | #fff8c5 (연황색) | --yellow |
| bearish | #ddf4ff (연파랑) | --down |

### VIX 색상 역전

VIX는 의미가 반대이므로 색상 역전:
- VIX 상승 --> 파랑(down) = 시장 불안
- VIX 하락 --> 빨강(up) = 시장 안도
- VIX >= 25 --> "공포 임계선 도달" 경고 배지

## 일간 리포트 (Daily Report)

**파일:** `src/lib/daily-html-builder.ts` (1,282줄)
**타이틀:** "일간 시장 분석 (M월 D일)"

### 섹션 구성 (7개)

```
+-----------------------------------------------+
| 헤더: 일간 시장 분석  [bullish|neutral|bearish] |
| YYYY-MM-DD · Market Analyst                    |
+-----------------------------------------------+
| 1. 시장 온도                                    |
|    Temperature Label + Rationale (LLM 텍스트)   |
+-----------------------------------------------+
| 2. 지수 현황                                    |
|    [SPY] [QQQ] [DJI] [RUT] [VIX]  카드 그리드   |
|    Fear & Greed: 점수 + 등급 + 추이             |
+-----------------------------------------------+
| 3. Phase 분포                                   |
|    [===P1===|====P2====|==P3==|=P4=] 가로 바    |
|    P2비율 | A/D | 신고/저가 | Breadth  칩 행    |
|    P2진입 | P2이탈 | 순유입  칩 행 (전환 통계)   |
|    LLM 해석 (breadthNarrative)                  |
+-----------------------------------------------+
| 4. 섹터 RS 랭킹                                 |
|    11개 섹터 테이블                               |
|    섹터 | RS | 순위 | 변동 | RS변화 | Phase | P2 |
+-----------------------------------------------+
| 5. 업종 RS Top 10                               |
|    상위 10개 업종 테이블 (섹터당 max 2개)         |
|    # | 업종 | 섹터 | RS | 4주변화 | Phase | P2  |
+-----------------------------------------------+
| 6. 특이종목                                     |
|    카드 그리드 (280px min)                       |
|    [TICKER | 업종 | 등락률%]                     |
|    [Phase | RS | 거래량비]                       |
|    [big_move] [high_volume] [phase_change] 태그 |
|    LLM 해석 (unusualStocksNarrative)            |
+-----------------------------------------------+
| 7. RS 상승 초기 종목 (0개면 생략)                |
|    종목 | Phase | RS | RS4주변화 | 업종 | 52w저점|
|    LLM 해석 (risingRSNarrative)                 |
+-----------------------------------------------+
| 풋터: Generated by Market Analyst               |
+-----------------------------------------------+
```

### 섹션별 데이터 상세

#### 1. 시장 온도

| 필드 | 출처 | 타입 |
|------|------|------|
| marketTemperature | LLM | "bullish" / "neutral" / "bearish" |
| marketTemperatureLabel | LLM | 예: "약세 -- 하락 3일째" |
| marketTemperatureRationale | LLM | 판단 근거 2~3문장 |

#### 2. 지수 현황

**도구:** `getIndexReturns(mode: 'daily')`

| 필드 | 설명 |
|------|------|
| symbol | ^GSPC, ^IXIC, ^DJI, ^RUT, ^VIX |
| name | S&P 500, NASDAQ, Dow Jones, Russell 2000, VIX |
| close | 종가 |
| changePercent | 당일 등락률 (%) |
| change | 전일 대비 절대값 |

**Fear & Greed:**

| 필드 | 설명 |
|------|------|
| score | 현재 점수 (0~100) |
| rating | Extreme Fear / Fear / Neutral / Greed / Extreme Greed |
| previousClose | 1주전 점수 |
| oneMonthAgo | 1달전 점수 |

**레이아웃:** 카드 그리드 `repeat(auto-fit, minmax(150px, 1fr))`

#### 3. Phase 분포

**도구:** `getMarketBreadth(mode: 'daily')`

| 필드 | 설명 | 표시 위치 |
|------|------|----------|
| totalStocks | 전체 종목 수 | 범례 |
| phase1Count~phase4Count | 각 Phase 절대 수 | 범례 + 바 |
| phase2Ratio | Phase 2 비율 (0~100, 이미 %) | 칩 |
| phase2RatioChange | 전일 대비 변화량 (%p) | 칩 (색상) |
| advancers / decliners | 상승/하락 종목 수 | A/D 칩 |
| newHighs / newLows | 52주 신고가/신저가 | 칩 |
| breadthScore | 종합 브레드스 스코어 | 칩 (있으면) |
| phase1to2Count1d | 당일 P1->P2 전환 수 | 칩 |
| phase2to3Count1d | 당일 P2->P3 전환 수 | 칩 |

**Phase 전환 칩 특수 로직:**
- P2 진입이 5일 평균 대비 1.5배 이상 --> 하이라이트(up 색상)
- 순유입 = P2진입 - P2이탈 (색상: 양수=up, 음수=down)

#### 4. 섹터 RS 랭킹

**도구:** `getLeadingSectors(mode: 'daily')`

| 컬럼 | 필드 | 색상 규칙 |
|------|------|----------|
| 섹터 | sector | -- |
| RS | avgRs | -- |
| 순위 | rsRank | -- |
| 순위변동 | rankChange | ▲ up / ▼ down / -- |
| RS변화 | rsChange | up/down |
| Phase | groupPhase | Phase 배지 |
| P2비율 | phase2Ratio | -- |

#### 5. 업종 RS Top 10

**도구:** `findTopIndustriesGlobal()` + `applyIndustrySectorCap(2, 10)`

| 컬럼 | 필드 | 비고 |
|------|------|------|
| # | 순위 | 1~10 |
| 업종 | industry | -- |
| 섹터 | sector | -- |
| RS | avgRs | 절대 RS 기준 정렬 |
| 4주변화 | change4w | up/down 색상 |
| Phase | groupPhase | Phase 배지 |
| P2비율 | phase2Ratio | -- |

**섹터캡:** 같은 섹터에서 최대 2개 업종만 포함 (집중도 완화)

#### 6. 특이종목

**도구:** `getUnusualStocks()` + 필터 `volRatio >= 1.0 && !splitSuspect`

| 필드 | 설명 |
|------|------|
| symbol | 종목 코드 |
| industry / sector | 업종 / 섹터 |
| dailyReturn | 당일 등락률 (%) |
| phase / prevPhase | 현재/이전 Phase |
| rsScore | RS 스코어 |
| volRatio | 거래량 비율 (당일 / VolMA30) |
| conditions[] | 조건 태그 배열 |

**조건 태그 색상:**

| 태그 | 배경 | 텍스트 | 의미 |
|------|------|--------|------|
| big_move | #ffebe9 | --up | 등락률 +-5% 이상 |
| high_volume | #ddf4ff | accent | 거래량 2x 이상 |
| phase_change | #fbefff | --purple | Phase 전환 발생 |
| phase2WithDrop | #fff3cd | --orange | P2인데 급락 |

**레이아웃:** 카드 그리드 `repeat(auto-fill, minmax(280px, 1fr))`

#### 7. RS 상승 초기 종목

**도구:** `getRisingRS()`

| 컬럼 | 필드 | 비고 |
|------|------|------|
| 종목 | symbol | -- |
| Phase | phase | Phase 배지 |
| RS | rsScore | RS 30~70 범위 |
| RS 4주변화 | rsChange | up/down 색상 |
| 업종 | industry | -- |
| 52w 저점비 | pctFromLow52w | -- |

**조건부 렌더링:**
- 0개 --> 섹션 전체 생략
- < 3개 --> 안내 텍스트: "(시장 상황에 따라 종목 수가 적을 수 있습니다)"

#### 8. 관심종목 현황

**도구:** `getWatchlistStatus(include_trajectory: false)`

**요약 칩 3개:**
- ACTIVE 종목 수
- 평균 P&L (up/down 색상)
- Phase 변화 종목 수

**테이블:**

| 컬럼 | 필드 | 비고 |
|------|------|------|
| 종목 | symbol | -- |
| 섹터 | entrySector | -- |
| 진입일 | entryDate | -- |
| 추적(일) | daysTracked | -- |
| Phase | currentPhase | Phase 배지 |
| 궤적(7일) | phaseTrajectory (최근 7개) | 도트 컴포넌트 |
| P&L | pnlPercent | up/down 색상 |
| RS | currentRsScore | -- |

**궤적 도트:**
- 22px 원형, Phase별 색상
- `title="YYYY-MM-DD"` 툴팁
- 최근 7일분만 표시 (일간 모드)

## 주간 리포트 (Weekly Report)

**파일:** `src/lib/weekly-html-builder.ts`
**타이틀:** "주간 시장 분석"

### 섹션 구성 (9개)

```
+-----------------------------------------------+
| 헤더: 주간 시장 분석  [bullish|neutral|bearish] |
| 3/30 ~ 4/4 · Market Analyst                   |

| 1. 주간 시장 구조 변화                           |

|    Fear & Greed: 점수 + 등급 + 1주/1달 비교     |
|    ----                                         |
|    Phase 분포 바 (주간 변화량 %p)                |
|    A/D | 신고/저가 | P1->P2(5일) | P2변화 칩행   |

|    11개 섹터 전체 테이블                          |

|    LLM 해석 (자금 집중 방향)                     |
| 4. 관심종목 궤적                                 |
|    [ACTIVE 수] [평균 P&L] [Phase변화] 칩 행     |
|    포트폴리오 편입 종목 제외 후 렌더링            |
|    종목(🔗)|섹터|진입일|추적|Phase|궤적(7일)|P&L|RS|SEPA|
|    🔗 = thesis 수혜주 시 hover 툴팁 (megatrend·bottleneck)|
|    LLM 해석 (서사 유효성)                        |
+-----------------------------------------------+
| 5. 포트폴리오 편입/탈락 + 현재 포트폴리오        |
|    5-A. 신규 편입/탈락 (LLM gate5 판단 기반)    |
|    5-B. 현재 포트폴리오 수익률 테이블            |
|         종목(🔗)|편입일|편입가|현재가|수익률|Phase |
|         🔗 = thesis 수혜주 시 hover 툴팁        |
+-----------------------------------------------+
| 6. 토론 인사이트                                 |
|    6-1. Thesis 충돌/강화 (LLM)                  |
|    6-2. 서사 체인 진화 (LLM)                     |
|    6-3. Thesis 적중률 피드백 (LLM)              |
+-----------------------------------------------+
| 7. 다음 주 관전 포인트                           |
|    7-1. 관전 포인트 (LLM)                       |
|    7-2. Thesis 시나리오 (LLM)                   |
+-----------------------------------------------+
| 8. 리스크 요인 (LLM)                            |
+-----------------------------------------------+
| 9. 레짐 맥락 (LLM)                              |
+-----------------------------------------------+
| 풋터: Generated by Market Analyst               |
+-----------------------------------------------+
```

### 일간과의 핵심 차이

| 측면 | 일간 | 주간 |
|------|------|------|
| **지수** | 종가 + 당일 변화% | 종가 + 주간 변화% + 주중 고/저 |
| **업종 정렬** | 절대 RS 기준 | **changeWeek** 기준 (자금 흐름 방향) |
| **Phase 분포** | 당일 스냅샷 | 주간 추이 (%p 변화량) |
| **관심종목 궤적** | 없음 | 전체 궤적 (`include_trajectory: true`) |
| **관심종목 SEPA** | 없음 | entrySepaGrade 컬럼 추가 |
| **관심종목 범위** | 없음 | 포트폴리오 편입 종목 제외 후 ACTIVE |
| **특이종목** | 있음 (카드 그리드) | 없음 |
| **RS 상승 초기** | 있음 (조건부) | 없음 |
| **포트폴리오** | 없음 | 편입/탈락 + 현재 수익률 테이블 |
| **토론 인사이트** | 없음 | 3개 소섹션 (Thesis 충돌/서사 진화/적중률) |
| **다음 주 포인트** | 없음 | 관전 포인트 + Thesis 시나리오 |
| **리스크/레짐** | 없음 (시장 온도에 포함) | 별도 섹션 |
| **LLM 필드 수** | 8개 | 14개 |

## UI 컴포넌트 라이브러리

### 1. Stat Chip (통계 칩)

```
+-------------------------+
| 라벨         stat-label |
| 값           stat-value |
| 부가 텍스트   stat-sub  |
+-------------------------+
```

- 배경: var(--surface)
- 테두리: 1px solid var(--border), radius 8px
- 패딩: 10px 16px
- Flex 행, gap 12px, flex-wrap
- **사용처:** Phase 2 비율, A/D, 신고/저가, 관심종목 요약

### 2. Index Card (지수 카드)

```
+------------------+
| 지수명    label  |
| 종가      value  |
| 등락률%   change |
| 전일대비         |
+------------------+
```

- 그리드: `repeat(auto-fit, minmax(150px, 1fr))`
- VIX: 색상 역전 + "공포 임계선" 배지 (>= 25)

### 3. Phase Bar (가로 스택 바)

```
[===P1(회색)===|====P2(보라)====|==P3(황)==|=P4(빨)=]
 P1: 25% (800)   P2: 42% (1,344)   P3: 20%   P4: 13%
```

- 높이: 24px, border-radius 6px
- 각 세그먼트: min-width 2%
- 범례: Phase별 비율(%) + 절대 수

### 4. Unusual Stock Card (특이종목 카드)

```
+-------------------------------+
| TICKER   업종/섹터   +5.23%  |
| Phase 2 · RS 75 · Vol 3.2x   |
| [big_move] [high_volume]      |
+-------------------------------+
```

- 그리드: `repeat(auto-fill, minmax(280px, 1fr))`
- 조건 태그: 각각 고유 배경/텍스트 색상

### 5. Trajectory Dot (궤적 도트)

```
[2] [2] [2] [3] [2] [2] [2]
 보라 보라 보라 황색 보라 보라 보라
```

- 22px 원형
- Phase 번호 표시
- 마우스호버 `title="YYYY-MM-DD"` 툴팁
- Phase별 배경색 적용

### 6. Phase Badge

```

```

## 주의사항

### phase2Ratio는 이미 % (0~100)

`phase2Ratio`는 도구 반환 시점에 이미 퍼센트 값이다.
**절대 x100 하지 않는다.** 이중 변환 시 2110%, 2330% 같은 버그 발생.
(`clampPercent()` 유틸로 범위 검증)

### 일간 vs 주간 업종 정렬 기준 차이

- **일간:** 절대 RS 기준 (`avgRs` 내림차순)
- **주간:** 주간 변화량 기준 (`changeWeek` 내림차순)
- 같은 "Top 10"이지만 의미가 다름. 일간은 "현재 강한 업종", 주간은 "이번 주 가장 많이 상승한 업종"

### LLM은 데이터 테이블을 생성하지 않는다

데이터 테이블은 100% 프로그래밍 렌더링.
LLM은 `content-block` 안의 **해석 텍스트만** 작성.
숫자 계산, 카운팅, 테이블 작성은 LLM에게 시키지 않는다.
