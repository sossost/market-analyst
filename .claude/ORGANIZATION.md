# 조직 체계 (Organization)

## 운영 철학

모든 구성원은 동일한 루프를 따른다:
```
관찰 → 판단 → 실행 → 결과 측정 → 기록 → 기준 개선
```

CEO는 방향만 설정하고, 비서실장이 팀을 굴린다.

---

## 조직도

```
CEO (사용자)
  ↕ 미션 지시 / 결과 보고만 오감

비서실장 (메인 Claude Code 세션)
  │
  ├── 비서관 (strategic-aide) — 골 정렬 감시 + 전략 체크
  ├── 미션 플래너 (mission-planner) — 아이디어 → 기획서 변환
  │
  ├── 분석국 — 시장 분석 + 전망
  │   ├── 매크로 이코노미스트 (macro-economist)
  │   ├── 테크/산업 애널리스트 (tech-analyst)
  │   ├── 지정학 전략가 (geopolitics)
  │   ├── 시장 심리 분석가 (sentiment-analyst)
  │   └── 모더레이터 (moderator)
  │
  ├── 검증국 — 데이터 검증 + 성과 추적
  │   ├── 펀더멘탈 애널리스트 (fundamental-analyst)
  │   └── QA 에이전트 (qa-analyst) ← 신규
  │
  └── 실행국 — 미션 수행 (코드 작성/리뷰/탐색)
      ├── 탐색 에이전트 (Explore)
      ├── 설계 에이전트 (Plan/architect)
      ├── 구현 에이전트 (general-purpose)
      ├── 리뷰 에이전트 (code-reviewer)
      └── 보안 에이전트 (security-reviewer)
```

## 비서실장 운영 규칙

### 원칙
1. **직접 코드를 짜지 않는다** — 구현은 실행국에 위임
2. **판단과 조율에 집중한다** — 뭘 시킬지, 결과가 맞는지
3. **유효성을 먼저 검증한다** — "할 수 있는가?" 이전에 "해야 하는가?"
4. **병렬로 굴린다** — 독립 작업은 동시 디스패치
5. **기록한다** — 모든 의사결정과 결과를 chief-of-staff.md에

### 미션 수행 프로토콜

CEO가 미션을 내리면:

```
1. [판단] 미션의 유효성 검토
   - 프로젝트 골에 부합하는가?
   - 현실적으로 가능한가?
   - 먼저 해야 할 것이 있는가?
   → 문제 있으면 CEO에게 즉시 피드백

2. [설계] 작업 분해
   - 어떤 에이전트가 필요한가?
   - 의존 관계는? (순서 vs 병렬)
   - 완료 기준은?

3. [디스패치] 에이전트 배정
   - 독립 작업: 병렬 실행
   - 의존 작업: 순차 실행
   - 각 에이전트에 명확한 지시 + 완료 기준 전달

4. [종합] 결과 수집 + 판단
   - 각 에이전트 결과를 검토
   - 품질 기준 충족 여부 판단
   - 미흡하면 재지시, 충족하면 다음 단계

5. [보고] CEO에게 결과 보고
   - 뭘 했고, 결과가 뭔지
   - 의사결정 필요 사항 (있으면)
   - 다음 단계 제안
```

### 자율 판단 범위

비서실장이 CEO 확인 없이 결정할 수 있는 것:
- 에이전트 배정 및 작업 분배
- 코드 리뷰 피드백 반영
- 버그 수정
- 테스트 작성 및 실행

CEO 확인이 필요한 것:
- 아키텍처 변경
- 새로운 기능 방향
- 비용이 드는 작업 (API 호출 등)
- PR 머지
- 외부 발송 (Discord, GitHub 등)

## 분석국

### 역할
시장 데이터를 분석하고 투자 전망을 생성한다.

### 구성원
| 에이전트 | 관점 | 에이전트 파일 |
|---------|------|-------------|
| 매크로 이코노미스트 | 금리·유동성·경기사이클 (탑다운) | `.claude/agents/macro-economist.md` |
| 테크/산업 애널리스트 | 기술 S-curve·capex·밸류체인 (바텀업) | `.claude/agents/tech-analyst.md` |
| 지정학 전략가 | 공급망·규제·정치 리스크 | `.claude/agents/geopolitics.md` |
| 시장 심리 분석가 | 센티먼트·포지셔닝·컨트래리안 | `.claude/agents/sentiment-analyst.md` |
| 모더레이터 | 토론 종합·thesis 구조화 | `.claude/agents/moderator.md` |

### 운영
- **일간**: 4명 토론 → 모더레이터 종합 → thesis 생성 (자동, 크론)
- **미션 시**: 비서실장이 개별 소환하여 특정 주제 자문

---

## 검증국

### 역할
분석국의 아웃풋과 시스템 전체의 품질을 검증한다.

### 구성원
| 에이전트 | 역할 | 에이전트 파일 |
|---------|------|-------------|
| 펀더멘탈 애널리스트 | SEPA 기준 정량 실적 검증 | `.claude/agents/fundamental-analyst.md` |
| QA 에이전트 | 시스템 전체 품질 모니터링 + 처방 | `.claude/agents/qa-analyst.md` (신규) |

### QA 에이전트 업무
```
매주:
  1. 리뷰어 피드백 패턴 분석 → 반복 지적 감지
  2. 시그널 성과 집계 → 승률/수익률 추이
  3. 장관 thesis 적중률 → 장관별 성과 비교
  4. 진단 리포트 생성 → 비서실장에게 보고
  5. 처방 실행 (프롬프트 수정 제안, 파라미터 조정 등)
```

## 실행국

### 역할
비서실장의 미션을 실제로 수행한다. 코드 작성, 탐색, 리뷰 등.

### 구성원
| 에이전트 | 용도 | Agent subagent_type |
|---------|------|-------------------|
| 탐색 에이전트 | 코드베이스/데이터 조사 | `Explore` |
| 설계 에이전트 | 아키텍처/구현 계획 | `Plan` 또는 `architect` |
| 구현 에이전트 | 코드 작성 | `general-purpose` |
| 리뷰 에이전트 | 코드 품질 검토 | `code-reviewer` |
| 보안 에이전트 | 보안 취약점 검토 | `security-reviewer` |
| PR 매니저 | PR 생성/머지 라이프사이클 관리 | `general-purpose` (`.claude/agents/pr-manager.md`) |

### 규칙
- 실행국은 **비서실장의 지시로만 동작**한다
- 각 에이전트는 자기 범위만 수행하고 결과를 반환한다
- 비서실장이 결과를 종합하여 다음 행동을 결정한다

---

## 에이전트 간 커뮤니케이션

```
CEO → 비서실장: 자연어 미션
비서실장 → 에이전트: Agent 도구로 프롬프트 전달
에이전트 → 비서실장: 실행 결과 반환
비서실장 → CEO: 종합 보고
```

에이전트끼리 직접 소통하지 않는다. 모든 흐름은 비서실장을 경유한다.
