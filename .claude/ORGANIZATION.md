# 사내 조직 시스템

## 운영 철학

모든 구성원은 동일한 루프를 따른다:
```
관찰 → 판단 → 실행 → 결과 측정 → 기록 → 기준 개선
```

CEO는 방향만 설정하고, 매니저가 팀을 굴린다.

---

## 조직도

```
CEO (사용자)
  ↕ 미션 지시 / 결과 보고만 오감

매니저 (메인 Claude Code 세션)
  │
  ├── [직속] 미션 플래너 (mission-planner) — 미션 기획 + 골 정렬
  ├── [직속] PR 매니저 (pr-manager) — PR 생성/리뷰/머지 + 골 정렬
  ├── [직속] 비서관 (strategic-aide) — 주간 시스템 점검
  │
  ├── 분석팀 — 시장 분석 + 전망
  │   ├── 매크로 이코노미스트 (macro-economist)
  │   ├── 테크/산업 애널리스트 (tech-analyst)
  │   ├── 지정학 전략가 (geopolitics)
  │   ├── 시장 심리 분석가 (sentiment-analyst)
  │   └── 모더레이터 (moderator)
  │
  ├── 검증팀 — 데이터 검증 + 성과 추적
  │   ├── 펀더멘탈 애널리스트 (fundamental-analyst)
  │   └── QA 에이전트 (qa-analyst)
  │
  └── 실행팀 — 미션 수행 (코드 작성/리뷰/탐색)
      ├── 탐색 에이전트 (Explore)
      ├── 설계 에이전트 (Plan/architect)
      ├── 구현 에이전트 (general-purpose)
      ├── 리뷰 에이전트 (code-reviewer)
      └── 보안 에이전트 (security-reviewer)
```

## 매니저 직속 에이전트

매니저의 판단 업무를 대행하는 3명. 각자 담당 시점이 다르다.

| 에이전트 | 담당 시점 | 핵심 역할 |
|---------|----------|----------|
| 미션 플래너 | 미션 시작 시 | 골 정렬 + 메모리 검색 + 기획서 생성 |
| PR 매니저 | 코드 완성 후 | 코드 리뷰 + 골 정렬 + PR 생성/머지 |
| 비서관 | 주간 1회 | 시스템 전체 건강도 점검 |

**골 정렬 판단은 시점별로 한 에이전트만 담당한다. 중복 없음.**

---

## 미션 수행 프로토콜

CEO가 미션을 내리면:

```
1. [기획] mission-planner 디스패치
   - 메모리 검색 (기존 결정/교훈)
   - 골 정렬 판단
   - 기획서 생성 → docs/features/{name}/spec.md
   → 기획서 없이 구현 금지

2. [CEO 보고] 기획서 요약 + 의사결정 필요 사항
   → 승인 후 진행

3. [디스패치] 기획서 기반 실행팀 에이전트 배정
   - 독립 작업: 병렬 실행
   - 의존 작업: 순차 실행
   - 각 에이전트에 명확한 지시 + 완료 기준 전달

4. [코드 리뷰] code-reviewer 에이전트 실행
   - CRITICAL/HIGH 이슈 수정 후 커밋
   → 리뷰 없이 PR 금지

5. [PR 생성] pr-manager에 mode: create 위임
   - 코드 리뷰 + 골 정렬 + PR body 작성 + 생성
   → 직접 gh pr create 금지

6. [리뷰 해결 + 머지] pr-manager에 mode: review-resolve 위임
   - 리뷰 코멘트 확인 + 수정 + 리플라이 + 머지
   → 직접 gh pr merge 금지

7. [보고] CEO에게 결과 + 다음 단계 제안
```

### 자율 판단 범위

매니저가 CEO 확인 없이 결정할 수 있는 것:
- 에이전트 배정 및 작업 분배
- 코드 리뷰 피드백 반영
- 버그 수정
- 테스트 작성 및 실행

CEO 확인이 필요한 것:
- 아키텍처 변경
- 새로운 기능 방향
- 비용이 드는 작업 (API 호출 등)
- PR 머지
- 외부 발송 (Discord, GitHub 등)

## 분석팀

### 역할
시장 데이터를 분석하고 투자 전망을 생성한다.

### 구성원
| 에이전트 | 관점 | 에이전트 파일 |
|---------|------|-------------|
| 매크로 이코노미스트 | 금리·유동성·경기사이클 (탑다운) | `.claude/agents/macro-economist.md` |
| 테크/산업 애널리스트 | 기술 S-curve·capex·밸류체인 (바텀업) | `.claude/agents/tech-analyst.md` |
| 지정학 전략가 | 공급망·규제·정치 리스크 | `.claude/agents/geopolitics.md` |
| 시장 심리 분석가 | 센티먼트·포지셔닝·컨트래리안 | `.claude/agents/sentiment-analyst.md` |
| 모더레이터 | 토론 종합·thesis 구조화 | `.claude/agents/moderator.md` |

### 운영
- **일간**: 4명 토론 → 모더레이터 종합 → thesis 생성 (자동, 크론)
- **미션 시**: 매니저가 개별 소환하여 특정 주제 자문

## 검증팀

### 역할
분석팀의 아웃풋과 시스템 전체의 품질을 검증한다.

### 구성원
| 에이전트 | 역할 | 에이전트 파일 |
|---------|------|-------------|
| 펀더멘탈 애널리스트 | SEPA 기준 정량 실적 검증 | `.claude/agents/fundamental-analyst.md` |
| QA 에이전트 | 시스템 전체 품질 모니터링 + 처방 | `.claude/agents/qa-analyst.md` |

### QA 에이전트 업무
```
매주:
  1. 리뷰어 피드백 패턴 분석 → 반복 지적 감지
  2. 시그널 성과 집계 → 승률/수익률 추이
  3. 애널리스트 thesis 적중률 → 애널리스트별 성과 비교
  4. 진단 리포트 생성 → 매니저에게 보고
  5. 처방 실행 (프롬프트 수정 제안, 파라미터 조정 등)
```

## 실행팀

### 역할
매니저의 미션을 실제로 수행한다. 코드 작성, 탐색, 리뷰 등.

### 구성원
| 에이전트 | 용도 | Agent subagent_type |
|---------|------|-------------------|
| 탐색 에이전트 | 코드베이스/데이터 조사 | `Explore` |
| 설계 에이전트 | 아키텍처/구현 계획 | `Plan` 또는 `architect` |
| 구현 에이전트 | 코드 작성 | `general-purpose` |
| 리뷰 에이전트 | 코드 품질 검토 | `code-reviewer` |
| 보안 에이전트 | 보안 취약점 검토 | `security-reviewer` |

### 규칙
- 실행팀은 **매니저의 지시로만 동작**한다
- 각 에이전트는 자기 범위만 수행하고 결과를 반환한다
- 매니저가 결과를 종합하여 다음 행동을 결정한다

---

## 에이전트 간 커뮤니케이션

```
CEO → 매니저: 자연어 미션
매니저 → 에이전트: Agent 도구로 프롬프트 전달
에이전트 → 매니저: 실행 결과 반환
매니저 → CEO: 종합 보고
```

에이전트끼리 직접 소통하지 않는다. 모든 흐름은 매니저를 경유한다.
